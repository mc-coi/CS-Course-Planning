**Why organize this way?**
- Related functions grouped together
- Easy to navigate and find code
- Follows professional conventions
- Scales well as project grows

### Version Control (Optional but Recommended)

After each working session, commit your progress:

```bash
# Initialize git (once)
git init

# After each day of work
git add .
git commit -m "Day 156: Project structure and constants"

# Good commit messages describe what changed
git commit -m "Day 157: Implemented get_scores() with validation"
git commit -m "Day 158: Added calculate_average() and unit tests"
```

## Guided Practice

### Set Up Your Project (45 min)

**Step 1: Create Main File (5 min)**
- Create `[project_name].py` in a dedicated folder
- Add module docstring (what the program does)
- Add your name and date

**Step 2: Add Structure (10 min)**
- Add section headers (INPUT, CALCULATION, OUTPUT, MAIN, etc.)
- Add all function stubs with docstrings (use `pass`)
- Add constants at the top

**Step 3: Test Structure (5 min)**
- Run the file — should have no errors
- Call `main()` — should do nothing (it's just `pass`)
- Good sign: no errors!

**Step 4: Test Foundational Functions (15 min)**
- Implement your 2–3 simplest functions
- Test each immediately
- Example: Test `calculate_average([85, 92, 78])` manually

**Step 5: Commit Progress (10 min)**
- Save all files
- If using git: `git add .` and `git commit -m "Day 156: Project setup"`
- Note what's done and what's next
