## Key Concepts

### Nearing Feature Completion

By end of Day 162, you should have:
- ✓ All core features (from Sprint 1)
- ✓ Additional features started (from Day 161)
- ✓ Additional features completed (today)
- ✓ All features integrated and tested

### Managing Scope

If you're running behind:
- Focus on critical features only
- Defer nice-to-haves to after Day 170
- Quality > quantity — make what you have excellent

If you're ahead:
- Add bonus features
- Refactor existing code
- Improve error handling
- Write unit tests

## Practice Problems

**Problem 1:** Create a checklist of all features you planned. What percentage is complete?

**Problem 2:** What's your single most important remaining feature? Why?

**Problem 3:** If you had to cut one feature to ship on time, which would it be? Why?

<details>
<summary>💡 Hints</summary>

- Problem 1: Divide done by total; if < 80%, you might need to cut scope
- Problem 2: Usually: one that many other features depend on
- Problem 3: Nice-to-have features; never critical path items

</details>
