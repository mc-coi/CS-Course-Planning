## Key Concepts

### What is the SDLC?

Professional software developers don't just open a text editor and start coding. They follow a **Software Development Lifecycle (SDLC)** — a structured process that ensures quality, reduces bugs, and makes large projects manageable.

The six phases:

| Phase | Focus | Output | Time |
|-------|-------|--------|------|
| **Define** | Understand the problem | Problem statement | Days 146–147 |
| **Plan** | Design the solution | Pseudocode, specs, flowcharts | Days 148–150 |
| **Design** | Create architecture | UI flow, data structures, function signatures | Days 151–155 |
| **Code** | Build it, test it | Working features | Days 156–165 |
| **Polish** | Refine, document, test | Polished, production-ready code | Days 166–172 |
| **Present** | Show your work | Presentation, demo, reflection | Days 173–180 |

### Why This Matters

**Without a process:**
- You write 100 lines of code, then realize you need to redesign everything
- You find bugs late (much harder to fix)
- Your code is messy and hard to understand
- You waste time on features that don't matter

**With a process:**
- You catch design problems *before* writing code
- Testing is built in from the start
- Your code is organized and maintainable
- You focus on the right features first

### The Waterfall vs. Iterative Approach

**Waterfall:** Complete each phase before moving to the next (rigid, but clear)
**Iterative:** Do mini-cycles of design → code → test → refine (flexible, feedback-driven)

For this project, we use **iterative design with two sprints** — you'll build core features, get feedback, then add more features. This is how real teams work.
