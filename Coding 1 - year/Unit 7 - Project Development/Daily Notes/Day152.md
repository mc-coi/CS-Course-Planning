# Day 152: Data Design — Planning Variables, Lists, and Structures

**Learning Target:** I can design data structures (variables, lists, dictionaries) that efficiently represent my program's data.

---

## Key Concepts

### What is Data Design?

Before you code, decide:
- What data do you need to store?
- What structure makes sense? (variable, list, dictionary, combination)
- How do you access it? (by name, by index, by key)
- How do you modify it? (append, update, delete)

**Good data design** makes your code simple and clear. **Bad data design** makes your code complicated and bug-prone.

### Data Structure Choices

**Single Variable:**
```python
student_name = "Alice"  # Simple value
student_age = 16
student_gpa = 3.85
```

**List:**
```python
scores = [85, 92, 78, 88]  # Multiple values of same type
students = ["Alice", "Bob", "Charlie"]
```

**Dictionary:**
```python
student = {
    "name": "Alice",
    "age": 16,
    "gpa": 3.85,
    "scores": [85, 92, 78, 88]
}
# Access: student["name"] → "Alice"
```

**List of Dictionaries:**
```python
students = [
    {"name": "Alice", "age": 16, "gpa": 3.85},
    {"name": "Bob", "age": 17, "gpa": 3.65},
]
# Access: students[0]["name"] → "Alice"
```

### Choosing the Right Structure

Ask these questions:

1. **Do I have one value or many?**
   - One → Variable
   - Many of same type → List
   - Many of different types → Dictionary

2. **Do I need to look up by name or by position?**
   - By position (0, 1, 2, ...) → List
   - By name (key) → Dictionary

3. **Will I add/remove data later?**
   - Yes → List or Dictionary
   - No → Variable might be enough

4. **Do I have multiple "records" (like students)?**
   - Yes → List of Dictionaries

## Code Examples

### Example 1: Grade Tracker Data Design

**Option A: Everything in separate lists (BAD)**
```python
names = ["Alice", "Bob"]
scores_alice = [85, 92, 78]
scores_bob = [90, 88, 92]

# Problem: Hard to add/remove students
# Problem: Unclear which scores belong to whom
```

**Option B: Dictionary of student → scores (GOOD)**
```python
students = {
    "Alice": [85, 92, 78],
    "Bob": [90, 88, 92]
}

# Add a student: students["Charlie"] = [88, 91, 89]
# Get Alice's scores: students["Alice"]
# Add a score to Alice: students["Alice"].append(95)
```

**Option C: List of student objects (BEST)**
```python
students = [
    {
        "name": "Alice",
        "scores": [85, 92, 78],
        "average": 85.0,
        "grade": "B"
    },
    {
        "name": "Bob",
        "scores": [90, 88, 92],
        "average": 90.0,
        "grade": "A"
    }
]

# Access: students[0]["name"] → "Alice"
# Access: students[0]["scores"] → [85, 92, 78]
# Add student: students.append({...})
# Remove student: students.pop(0)
```

### Example 2: Todo App Data Design

```python
# Option 1: Simple list of task strings (BASIC)
tasks = ["Buy groceries", "Finish homework", "Call Mom"]
# Problem: Can't track status, due date, priority

# Option 2: List of task dictionaries (GOOD)
tasks = [
    {
        "description": "Buy groceries",
        "status": "incomplete",  # or "complete"
        "due_date": "2026-03-30",
        "priority": "high"
    },
    {
        "description": "Finish homework",
        "status": "complete",
        "due_date": "2026-03-28",
        "priority": "high"
    }
]

# Access first task: tasks[0]
# Get status of first task: tasks[0]["status"]
# Add task: tasks.append({...})
# Mark complete: tasks[0]["status"] = "complete"
# Filter complete tasks: [t for t in tasks if t["status"] == "complete"]
```

### Example 3: Data Structure Design Document

```
PROJECT: Grade Tracker
======================

DATA STORAGE:

1. Student Information
   Type: Dictionary (key = name, value = grade data)
   Structure:
   {
     "Alice": {
       "scores": [85, 92, 78],
       "average": 85.0,
       "grade": "B"
     },
     "Bob": {
       "scores": [90, 88, 92],
       "average": 90.0,
       "grade": "A"
     }
   }

   Why: Easy to look up student by name
        Easy to add/remove students
        Easy to store multiple related data per student

2. Individual Score Entry
   Type: List of integers/floats
   Example: [85, 92, 78]
   Why: Simple, familiar from Unit 4
        Easy to calculate average with sum() and len()

3. Report (temporary, for display)
   Type: Dictionary of strings
   Example:
   {
     "name": "Alice",
     "scores": "85, 92, 78",
     "average": "85.0",
     "grade": "B",
     "status": "PASSING"
   }
   Why: Easy to format and print
        Easy to save to file

OPERATIONS:

Add Student:
  students[new_name] = {"scores": [], "average": 0, "grade": "N/A"}

Add Score:
  students[name]["scores"].append(score)
  students[name]["average"] = calculate_average(students[name]["scores"])
  students[name]["grade"] = assign_grade(students[name]["average"])

Remove Student:
  del students[name]

Get Student's Average:
  average = students[name]["average"]

List All Students:
  for name in students:
      print(name)

FILE I/O:
  Format: One line per student, tab-separated
  Example: Alice	85,92,78	85.0	B
  Read: Split each line, rebuild dictionary
  Write: For each student, write name, scores, avg, grade
```

## Guided Practice

### Design Your Data Structures (45 min)

**Step 1: Identify What Data You Need (10 min)**
- What information does your program track?
- Example (Todo app): description, status, due_date, priority
- Example (Grade tracker): name, scores, average, grade

**Step 2: Choose Structures (15 min)**
- For each piece of data, choose: variable, list, dictionary, or combination
- Ask: "How do users interact with this data?"
- Draw diagrams if it helps

**Step 3: Write a Data Structure Document (15 min)**
- Describe each structure in plain English
- Show an example with real data
- Explain why you chose this structure
- List the operations (add, remove, access, calculate)

**Step 4: Code a Sample (5 min)**
- Create sample data for your project
- Include 2-3 examples
- Example:
  ```python
  # Sample data for Grade Tracker

      "Bob": {"scores": [90, 88, 92], "average": 90.0, "grade": "A"}
  }
  ```
