## Key Concepts

### Making Progress on a Work Day

**Structure of a work day (55 min):**
- 10 min: Check-in, review yesterday, plan today
- 30 min: Code and test (focus mode)
- 10 min: Fix any issues that came up
- 5 min: Commit progress, note blockers

### Staying Focused

**During focus time:**
- Code ONE function or feature at a time
- Test it immediately after
- Fix bugs before moving on
- Don't skip to the next feature

**Avoid:**
- Context switching (don't switch between features)
- Writing tons of code then testing (test frequently!)
- Leaving bugs for "later"

When code doesn't work:

def calculate_average(scores):
    print(f"[DEBUG] Received scores: {scores}")
    if not scores:
        return 0.0
    total = sum(scores)
    print(f"[DEBUG] Sum: {total}, Count: {len(scores)}")

    print(f"[DEBUG] Average: {avg}")

### Common Bugs to Watch For

**Type Errors:**
```python
# ✗ Wrong
score = input("Score: ")  # This is a STRING
total = score + 100  # Can't add string + number!

# ✓ Correct
score = float(input("Score: "))
total = score + 100
```

**Off-by-One Errors:**
```python
# ✗ Wrong
for i in range(len(scores)):
    print(scores[i])  # Fine unless scores is changing size

# ✓ Correct (more robust)
for score in scores:
    print(score)
```

**Logic Errors:**
```python
# ✗ Wrong
if average >= 90:
    grade = "A"
elif average >= 90:  # This will never run!
    grade = "B"

# ✓ Correct
if average >= 90:
    grade = "A"
elif average >= 80:  # Different condition
    grade = "B"
```

## Mini-Challenge: Rubber Duck Debugging

When you're stuck, explain your code to an imaginary rubber duck:

```python
def get_scores():
    scores = []
    while True:
        entry = input("Score (or 'done'): ")
        # Duck, here's what I'm doing:
        # I ask the user for a score
        # If they say 'done', I should stop
        if entry == "done":
            break
        # Otherwise, I convert to float and add to list
        score = float(entry)
        scores.append(score)
    return scores

# Duck says: "But what if they type 'done' without entering a score first?"
# Me: "Oh! I should check if scores is empty before returning"
# Fixed version:
def get_scores():
    scores = []
    while True:
        entry = input("Score (or 'done'): ")
        if entry == "done":
            if scores:  # ← Added this check
                break
            else:
                print("Enter at least one score")
                continue
        score = float(entry)
        scores.append(score)
    return scores
```
