# Day 166: Code Cleanup — Comments, Docstrings, and Naming

**Learning Target:** I can improve code readability by adding clear comments, complete docstrings, and descriptive variable names.

---

## Key Concepts

### Three Levels of Code Documentation

**1. Docstrings (Function Documentation)**
```python
def calculate_average(scores):
    """
    Calculate the arithmetic mean of a list of scores.

    Parameters:
        scores (list of float): Scores to average

    Returns:
        float: The mean (0.0 if list is empty)
    """
    if not scores:
        return 0.0
    return sum(scores) / len(scores)
```

**2. Inline Comments (Explain "Why")**
```python
def assign_grade(average):
    """Convert average to letter grade A-F."""
    # Grades: A (90+), B (80-89), C (70-79), D (60-69), F (<60)
    if average >= 90:
        return "A"
    elif average >= 80:
        return "B"
    # ... etc
```

**3. Variable Names (Self-Documenting)**
```python
# ✗ Bad
t = 0
for i in x:
    t += i
a = t / len(x)

# ✓ Good
total = 0
for score in scores:
    total += score
average = total / len(scores)
```

### Cleanup Checklist

- [ ] All functions have docstrings
- [ ] Variable names are descriptive (not x, temp, stuff)
- [ ] Complex logic has inline comments
- [ ] No dead code (commented-out lines removed)
- [ ] Module has docstring at top
- [ ] Consistent indentation (4 spaces)

## Guided Practice

### Code Cleanup (Full Period)

**Step 1: Add Module Docstring (5 min)**
```python
"""
Grade Tracker
A program for students to track test scores, calculate averages, and generate reports.

Author: [Your name]
Date: [Today]
"""
```

**Step 2: Review Function Docstrings (15 min)**
- Check every function
- If missing, add one
- Use standard format: purpose, parameters, returns

**Step 3: Improve Variable Names (15 min)**
- Find vague names (x, temp, stuff, data)
- Replace with descriptive names
- Examples:
  - `x` → `score`
  - `temp` → `average`
  - `f` → `student_file`

**Step 4: Add Inline Comments (15 min)**
- Find complex logic
- Explain "why," not "what"
- Example:
  ```python
  # Check if average is 89.9+ to catch rounding edge cases
  if average >= 89.95:
      grade = "A"
  ```

**Step 5: Remove Dead Code (5 min)**
- Delete commented-out lines
- If you need it again, git will have it!
