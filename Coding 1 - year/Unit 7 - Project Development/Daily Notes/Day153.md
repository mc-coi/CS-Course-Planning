# Day 153: Function Design — Planning Function Signatures

**Learning Target:** I can plan and document function signatures (name, parameters, return values) before implementing them.

---

## Key Concepts

### Function Signature

A **function signature** is its "contract":
- **Name:** What does it do?
- **Parameters:** What data does it need?
- **Return value:** What does it produce?
- **Docstring:** What does it do?

```python
def calculate_average(scores):
    """
    Calculate and return the arithmetic mean of a list of scores.

    Parameters:
        scores (list of float): List of scores to average

    Returns:
        float: The arithmetic mean, or 0.0 if list is empty
    """
    if not scores:
        return 0.0
    return sum(scores) / len(scores)
```

### Good Function Design Principles

**Single Responsibility:** Each function does one thing
- `get_scores()` — gets input
- `validate_score(score)` — validates one score
- `calculate_average(scores)` — calculates stats
- `assign_grade(average)` — converts to letter

**Clear Parameters:** Each parameter has a clear meaning
- ✅ `get_scores_from_user()` — clear
- ✗ `get_data()` — vague

**Meaningful Return:** Return value matches what you need
- ✅ `calculate_average(scores)` returns float
- ✗ `calculate_average(scores)` returns string

**Handles Edge Cases:** Think about unusual inputs
- Empty list? Negative numbers? Zero?

## Guided Practice

### Design Your Function Signatures (45 min)

**Step 1: List All Functions (10 min)**
- From your pseudocode (Day 148), what functions do you need?
- Write them down with one-line descriptions
- Example: `add_task()`, `display_tasks()`, `mark_complete()`, etc.

**Step 2: For Each Function, Decide (20 min)**
- What does it do? (Write a clear docstring)
- What does it need? (Parameters)
- What does it produce? (Return value)
- What edge cases must it handle?

**Step 3: Create a Function Design Document (15 min)**
- Use the table format from Example 2
- Include all your functions
- For each: name, purpose, parameters, return type

**Step 4: Code the Skeletons (5 min)**
- Create a Python file with all function definitions
- Include docstrings
- Use `pass` as the body (don't implement yet)
- Example:
  ```python
  def add_task(task_list, description):
      """Add a new task to the list."""
      pass
  ```
