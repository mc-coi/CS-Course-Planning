# Day 167: Systematic Testing — Edge Cases and Stress Testing

**Learning Target:** I can write comprehensive test cases that exercise my program thoroughly and identify remaining bugs.

---

## Key Concepts

### Types of Test Cases

**Normal Cases:**
```python
# Grade Tracker: Normal case
Input: Alice, [85, 92, 78]
Expected: Average 85.0, Grade B
Result: ✓ Pass
```

**Edge Cases (Boundary Conditions):**
```python
# Empty list
Input: Alice, []
Expected: Average 0.0, Grade F, error message
Result: ✓ Pass

# Single score
Input: Alice, [85]
Expected: Average 85.0, Grade B
Result: ✓ Pass

# All same score
Input: Alice, [80, 80, 80]
Expected: Average 80.0, Grade B
Result: ✓ Pass

# Boundary grades
Input: Alice, [89.5]  # Just below A
Expected: Grade B
Result: ✓ Pass

Input: Alice, [90.0]  # Exactly A
Expected: Grade A
Result: ✓ Pass
```

**Error Cases:**
```python
# Invalid input
Input: Bob, "abc" (instead of scores)
Expected: Error message, ask again
Result: ✓ Pass

# File doesn't exist
Input: Load from nonexistent.txt
Expected: Error message, continue
Result: ✓ Pass

# Permission denied (if applicable)
Result: ✓ Graceful error
```

### Test Case Template

```
═══════════════════════════════════════════════════════════
TEST CASE #1: Normal Grade Calculation

Scenario: Enter three normal scores

  Student name: Alice
  Scores: 85, 92, 78

Expected Output:
  Average: 85.0
  Grade: B
  Status: PASSING

Actual Output:
  Average: 85.0
  Grade: B
  Status: PASSING

Result: ✓ PASS

═══════════════════════════════════════════════════════════
TEST CASE #2: Edge Case — Empty Scores

Scenario: Student enters no scores
Input:
  Student name: Bob
  Scores: [type 'done' immediately]

Expected Output:
  Error message: "You must enter at least one score"
  Program asks again

Actual Output:
  Error message shown ✓
  Loop repeats ✓

Result: ✓ PASS

═══════════════════════════════════════════════════════════
TEST CASE #3: Error Case — Invalid File

Scenario: Try to load from file that doesn't exist
Input:
  Load from: nonexistent_file.txt

Expected Output:
  Error message: "File not found"
  Program uses default data

Actual Output:
  [Write what actually happened]

Result: [✓ PASS or ✗ FAIL]
If FAIL: Describe bug and fix

═══════════════════════════════════════════════════════════
```

## Guided Practice

### Systematic Testing (Full Period)

**Step 1: List Test Cases (10 min)**
- Normal cases (1-2)
- Edge cases (3-5)
- Error cases (2-3)
- Total: ~8-10 test cases

**Step 2: Create Test Plan (10 min)**
- Write out each test case
- Note expected output
- Prepare test inputs (write them down!)

**Step 3: Run Tests (25 min)**
- Execute each test case
- Record actual output
- Mark as PASS or FAIL
- If FAIL, note the bug

**Step 4: Fix Bugs (5 min)**
- For any FAILs, identify the problem
- Fix the code
- Re-test that case

**Step 5: Document Results (5 min)**
- Save test results
- Commit with message: "Day 167: Comprehensive testing, [X] of [Y] tests pass"

Expected: Avg 89.5, Grade B (just below 90)

TEST CASE 6: Edge — exact boundary
Input: "Frank", [90.0]

Status: ✓ PASS

TEST CASE 7: Error — invalid input
Input: "Grace", "abc" (instead of number)
Expected: Error message, ask again
Status: ✓ PASS

TEST CASE 8: Error — negative number
Input: "Henry", [-10]
Expected: Error or rejection, ask again
Status: [Check: does your program handle this?]

TEST CASE 9: Error — file doesn't exist on load
Expected: Error message, use default
Status: ✓ PASS

TEST CASE 10: Normal — save and load
Input: Save "test_report.txt"
Expected: File created, can load it back
Status: ✓ PASS

SUMMARY: 9/10 tests pass
Failing test: Test Case 8 (negative numbers)
  → Bug: No validation for negative scores
  → Fix: Add check: if score < 0, reject
