# Day 174: Presentation Rehearsal with Peer Feedback

**Learning Target:** I can deliver my presentation smoothly and incorporate feedback to improve it.

---

## Key Concepts

### Presentation Best Practices

**Do:**
- Make eye contact with audience
- Speak clearly and slowly
- Pause when you're thinking
- Let audience process information
- Be confident (own your work!)

**Don't:**
- Read slides word-for-word
- Talk too fast
- Apologize for bugs (own them instead)
- Memorize everything (talk naturally)

### Handling Questions

If asked a question:
1. Listen carefully
2. Ask for clarification if needed
3. Answer honestly
4. Say "I don't know" if unsure (better than guessing)

---

## Guided Practice

### Full Rehearsal & Feedback (Full Period)

**Phase 1: First Run-Through (15 min)**
- Find a peer or teacher
- Present from start to finish (no stopping)
- Time yourself
- Record or have peer take notes

**Phase 2: Peer Feedback (5 min)**
- Peer gives feedback using checklist
- What was clear?
- What was confusing?
- Any suggestions?

**Phase 3: Refinement (15 min)**
- Address feedback
- Practice any tricky parts
- Adjust timing if needed
- Second run-through

**Phase 4: Final Polish (15 min)**
- Make final adjustments
- Practice opening and closing (you'll be nervous)
- Prepare demo one more time
- Feel confident

**Phase 5: Debrief (5 min)**
- How do you feel about your presentation?
- What will you change for the real thing?
- Are you ready?

## Feedback Checklist

When your peer presents:

```
PRESENTATION FEEDBACK

Presenter: [Name]
Reviewer: [You]

CLARITY:
[✓] Introduction was clear
[✓] Problem was well-explained
[✓] Demo showed features effectively
[✓] Code walkthrough was understandable
[  ] Needs improvement: [describe]

PACING:
[✓] Good pace (not too fast/slow)
[✓] Good pauses (gave audience time to process)
[  ] Too fast in: [section]
[  ] Too slow in: [section]

ENGAGEMENT:
[✓] Made eye contact
[✓] Spoke with confidence
[✓] Energy was good
[  ] Could improve: [describe]

TECHNICAL:
[✓] Demo worked
[✓] Slides were visible
[✓] Audio was clear
[  ] Issue: [describe]

OVERALL: 9/10 — Great job!

Strengths:
- Clear explanation of the problem
- Live demo was impressive
- Good command of your code

Suggestions:
1. Speak a bit slower in the walkthrough section
2. Could emphasize the error handling more
3. Great job overall — you're ready!
```
