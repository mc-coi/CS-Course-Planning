## Guided Practice

### Your Sprint 2 Retrospective (Full Period)

**Step 1: Assess Sprint 2 (10 min)**
- What features are complete?
- What works well?
- What needs improvement?

**Step 2: Evaluate Code Quality (10 min)**
- Read through your code
- Check docstrings (all present?)
- Check comments (enough explanation?)
- Check error handling (robust?)

**Step 3: Identify Polish Tasks (10 min)**
- What code cleanup is needed?
- What testing should you do?
- What bugs remain?
- What UX improvements would help?

**Step 4: Plan Polish Phase (15 min)**
- Prioritize improvements
- Estimate time for each
- Decide what must vs. should happen

**Step 5: Document & Commit (10 min)**
- Write retrospective
- Commit with message: "Day 165: Sprint 2 complete, retrospective done"
