# Day 149: Breaking Projects into Milestones and Tasks

**Learning Target:** I can decompose my project into milestones and tasks with realistic timelines.

---

## Key Concepts

### Milestones vs. Tasks

**Task:** One specific, completable piece of work (usually 1-2 hours)
- Example: "Implement the `get_scores()` function"
- Example: "Test `calculate_average()` with 5 test cases"

**Milestone:** A significant completion point (usually 1-2 days)
- Example: "All core functions written and unit tested"
- Example: "MVP complete and integrated"

**Project:** A collection of milestones over weeks
- Example: Grade Tracker (5 weeks)

### Breaking Down Your Project

Good decomposition is **crucial**. It helps you:
- Track progress clearly
- Know what to work on next
- Estimate time realistically
- Celebrate small wins
- Identify bottlenecks early

## Guided Practice

### Create Your Project Task List (45 min)

Using your pseudocode from Day 148, break down your project:

**Step 1: List All Functions (10 min)**
- From your pseudocode, identify every function you need to write
- Example: `get_student_name()`, `get_scores()`, `main()`, etc.

**Step 2: Organize by Milestone (15 min)**
- Group functions into the 7 milestones (design, prototype, sprint 1, sprint 2, polish, documentation, presentation)
- Ask: "Which functions are foundational? Which depend on others?"
- Example: `calculate_average()` should be done before `assign_grade()`

**Step 3: Break Into Daily Tasks (15 min)**
- For each milestone, estimate how many days it needs
- Create a task list (checkbox format) for each milestone
- Be realistic — one function per day is reasonable

**Step 4: Identify Dependencies (5 min)**
- What must be done before what?
- Example: You can't test `main()` until helper functions exist
- Mark these as "blockers"
