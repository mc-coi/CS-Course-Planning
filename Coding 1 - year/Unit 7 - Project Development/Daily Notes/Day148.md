# Day 148: Planning with Pseudocode and Flowcharts

**Learning Target:** I can write detailed pseudocode and draw flowcharts that outline my program's logic before coding.

---

## Key Concepts

### What is Pseudocode?

**Pseudocode** is an outline of your program written in plain English (not Python syntax). It's detailed enough to code from, but simple enough to plan with.

**Why use it?**
- Separates logic from syntax (focus on "what" before "how")
- Easy to spot design problems before investing in code
- Serves as documentation alongside your code
- Makes coding much faster (you have a roadmap)

### Writing Good Pseudocode

**Characteristics:**
- Written in plain English
- Numbered steps
- Shows flow (if/then, loops)
- Each line is one logical action
- Specific (not vague)
- Matches your code structure exactly

### Flowcharts

A **flowchart** is a visual representation of your program's logic.

**Symbols:**
- **Oval/Circle:** Start/End
- **Rectangle:** Process (do something)
- **Diamond:** Decision (yes/no question)
- **Parallelogram:** Input/Output
- **Arrows:** Flow direction

## Code Examples

### Example 1: Pseudocode for Grade Tracker

```
PROGRAM: Grade Tracker v1

1. Display welcome message
2. FUNCTION get_student_name()
   a. Prompt user for student name
   b. If name is empty, ask again
   c. Return name
3. FUNCTION get_scores()
   a. Initialize empty scores list
   b. LOOP until user types "done":
      i. Prompt for a score
      ii. TRY to convert to float
          - If invalid, show error and continue loop
      iii. IF score is between 0-100:
          - Add to scores list
      iv. ELSE:
          - Show error message and continue loop
   c. Return scores list
4. FUNCTION calculate_average(scores)
   a. IF scores list is empty:
      - Return 0
   b. ELSE:
      - Calculate sum of all scores
      - Divide by count of scores
      - Return average
5. FUNCTION assign_grade(average)
   a. IF average >= 90: return "A"
   b. ELSE IF average >= 80: return "B"
   c. ELSE IF average >= 70: return "C"
   d. ELSE IF average >= 60: return "D"
   e. ELSE: return "F"
6. FUNCTION display_report(name, scores, avg, grade)
   a. Print formatted header
   b. Print student name
   c. Print list of all scores
   d. Print average
   e. Print assigned grade
   f. Print footer
7. FUNCTION save_to_file(filename, report_text)
   a. TRY to open file for writing
      - Write report_text to file
      - Close file
      - Show success message
   b. CATCH IOError:
      - Show error message
8. MAIN program:
   a. Call display_welcome()
   b. name = get_student_name()
   c. scores = get_scores()
   d. avg = calculate_average(scores)
   e. grade = assign_grade(avg)
   f. Call display_report(name, scores, avg, grade)
   g. Ask user: "Save to file? (y/n)"
   h. IF user says "yes":
      - filename = create filename from name
      - save_to_file(filename, report_text)
   i. Print thank you message
```

### Example 2: Simpler Pseudocode (Text Adventure Entry)

```
PROGRAM: Simple Room Explorer

1. DISPLAY opening description
2. CREATE map of rooms as dictionary
   rooms = {
     "Hall": {"description": "...", "items": [...], "exits": {...}},
     ...
   }
3. CREATE empty inventory list
4. current_room = "Hall"
5. LOOP until user types "quit":
   a. DISPLAY current room's description
   b. DISPLAY available exits
   c. GET user command
   d. PARSE command:
      - IF "go [direction]":
         - Check if direction is valid exit
         - IF valid: move to new room
         - ELSE: show error
      - IF "take [item]":
         - Check if item exists in room
         - IF exists: add to inventory, remove from room
         - ELSE: show error
      - IF "inventory":
         - Display items in inventory
      - IF "quit":
         - Exit loop
```

## Guided Practice

### Convert Your Proposal to Pseudocode (45 min)

Take your project proposal from Day 147. Convert it into detailed pseudocode:

**Step 1: List Your Functions (10 min)**
- From your proposal's core features, what functions do you need?
- Write them down with brief descriptions

**Step 2: Outline Main() (5 min)**
- What happens when the program runs?
- In what order do you call functions?
- Do you loop? Do you ask for user input at the end?

**Step 3: Detailed Pseudocode (30 min)**
- For each function, write numbered steps in plain English
- Show conditionals (IF/ELSE)
- Show loops (LOOP UNTIL / WHILE)
- Show input/output
- Use actual variable names

**Step 4: Get Feedback (5 min)**
- Ask a peer: "Could you code this without asking me questions?"
- If they ask clarifying questions, your pseudocode needs more detail!

```python
# Option A
def get_scores():
    scores = []
    while True:
        entry = input("Score (or 'done'): ")
        if entry.lower() == "done":
            break
        try:
            score = float(entry)
            if 0 <= score <= 100:
                scores.append(score)
        except ValueError:
            print("Invalid input")
    return scores

def get_scores():
    scores = []
    for i in range(10):
```
