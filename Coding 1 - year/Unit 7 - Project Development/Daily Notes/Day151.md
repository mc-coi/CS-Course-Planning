## Key Concepts

### What is UI Design?

**UI (User Interface)** is how your program talks to users. For a text-based program, it's:
- Menus (what choices do they see?)
- Prompts (what are you asking them to do?)
- Output (how do you display results?)
- Flow (in what order do things happen?)

Good UI is:
- **Clear:** Users understand what they can do
- **Logical:** Options flow naturally
- **Forgiving:** Wrong input doesn't crash the program
- **Responsive:** Program acknowledges user actions

### Text Menu Patterns

**Pattern 1: Simple Menu**
```
===== Grade Tracker =====
1. Add student
2. View report
3. Exit
Choose (1-3):
```

**Pattern 2: Nested Menus**
```
===== Main Menu =====
1. Manage students
2. Manage grades
3. Exit
Choose: 1

===== Manage Students =====
1. Add student
2. Delete student
3. Back to main menu
Choose:
```

**Pattern 3: Input → Process → Display**
```
Enter student name: Alice
Enter scores (type 'done' when finished):
Score 1: 85
Score 2: 92
Score 3: 78
Score 4: done

Processing...
Average: 85.0
Grade: B
```

### Designing Interaction Flow

Map out **every** user choice and what happens next.

Example: Grade Tracker flow
```
START
↓
Show main menu
↓
User chooses:
├─ (1) Add → Add student name → Return to menu
├─ (2) View grades → Select student → Show report → Return to menu
├─ (3) Save → Select format → Save file → Confirm
└─ (4) Exit → Show "Thank you" → END
```

## Code Examples

### Example 1: Good Menu Design (Clear, Documented)

```python
"""
Grade Tracker - User Interface Design
Interaction flow is documented below.
"""

def show_main_menu():
    """Display main menu and return user's choice."""
    print("\n" + "=" * 40)
    print("    GRADE TRACKER - Main Menu")
    print("=" * 40)
    print("1. Add a student")
    print("2. View student report")
    print("3. Compare students")
    print("4. Save all data to file")
    print("5. Exit")
    print("=" * 40)

    while True:
        choice = input("\nEnter choice (1-5): ").strip()
        if choice in ["1", "2", "3", "4", "5"]:
            return choice
        print("Invalid choice. Please enter 1-5.")


def show_student_menu():
    """Display menu for managing a student."""
    print("\n" + "-" * 40)
    print("    What would you like to do?")
    print("-" * 40)
    print("1. Add a score")
    print("2. View grades")
    print("3. Remove a score")
    print("4. Back to main menu")
    print("-" * 40)

    while True:
        choice = input("\nEnter choice (1-4): ").strip()
        if choice in ["1", "2", "3", "4"]:
            return choice
        print("Invalid choice. Please enter 1-4.")


def get_valid_input(prompt, input_type=str, constraints=None):
    """
    Get user input with validation.

    Args:
        prompt (str): What to ask the user
        input_type: Type to convert to (int, float, str)
        constraints: Function that returns True if valid (e.g., lambda x: 0 <= x <= 100)

    Returns:
        Validated input in specified type
    """
    while True:
        try:
            user_input = input(prompt).strip()
            if not user_input:
                print("Input cannot be empty.")
                continue

            converted = input_type(user_input)

            if constraints and not constraints(converted):
                print(f"Invalid input. Please try again.")
                continue

            return converted
        except ValueError:
            print(f"Invalid input. Please enter a valid {input_type.__name__}.")
```

    ├─ Invalid input (e.g., letters instead of numbers) → Show error, ask again

## Guided Practice

### Design Your UI (45 min)

**Step 1: Main Menu (10 min)**
- What are the top-level options?
- Write them out as a menu
- Example:
  ```
  ===== My Todo App =====
  1. Add task
  2. View tasks
  3. Mark complete
  4. Exit
  ```

**Step 2: Submenus (10 min)**
- For each main option, is there a submenu?
- Example: If user selects "View tasks," do they choose:
  - All tasks?
  - Complete tasks?
  - Incomplete tasks?

**Step 3: Input Prompts (10 min)**
- For each action, what do you need to ask the user?
- Write the exact prompts
- Example: "Enter task description: " not just "Task: "

**Step 4: Output Display (10 min)**
- How will results be formatted?
- Make it readable: use borders, alignment, consistent spacing
- Example:
  ```
  ===== Task Report =====
  Task: Buy groceries
  Status: [  ] Not Done
  Due Date: 2026-03-30
  ```

**Step 5: Draw a Flow Diagram (5 min)**
- Show how users move through menus
- Indicate what happens at each choice
