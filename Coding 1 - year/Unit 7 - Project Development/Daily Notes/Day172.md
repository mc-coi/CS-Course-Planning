# Day 172: Code Walkthrough Practice

**Learning Target:** I can explain my code clearly, discussing design choices and key functions.

---

## Key Concepts

### Code Walkthrough

A **code walkthrough** means explaining your code to someone else:
- What does each function do?
- Why did you design it that way?
- How do the pieces fit together?

**Not:**
- Reading code line-by-line
- Reciting what the code does
- Explaining syntax

**Yes:**
- Explaining logic and purpose
- Discussing tradeoffs
- Showing interesting/complex parts

---

## Example Code Walkthrough Script

```
"I'm going to walk you through the most interesting part of my
code — the input validation.

This function, get_scores(), collects scores from the user. The
tricky part is validation. Here's what it does:

1. Loop until the user says 'done'
2. For each entry, try to convert to float
3. If conversion fails (they type letters), catch ValueError
4. If conversion succeeds, check if it's 0-100
5. If in range, add it; otherwise ask again

Why this design? The error handling is important because users
might type 'abc' by mistake. Instead of crashing, the program
just asks again. The range check prevents impossible scores.

Here's the code... [shows code] ...the key part is the try/except.
Without it, this line would crash: `score = float(entry)`. With
it, we catch the error and give the user a chance to fix it.

Any questions about how this works?"
```

## Guided Practice

### Practice Your Walkthrough (Full Period)

**Step 1: Choose 1-2 Key Functions (5 min)**
- Pick your most interesting functions
- Usually: input validation, calculation, file I/O

**Step 2: Prepare Explanation (15 min)**
- Why is this function important?
- How does it work?
- What edge cases does it handle?
- Why did you design it this way?

**Step 3: Write a Script (10 min)**
- Write your walkthrough as a script
- Keep it to 1-2 minutes per function
- Focus on logic, not syntax

**Step 4: Practice Out Loud (15 min)**
- Read your script aloud
- Time yourself
- Adjust if too long/short
- Practice until smooth

**Step 5: Get Feedback (10 min)**
- Ask a peer or teacher to listen
- Get feedback on clarity
- Revise based on feedback

---

## Practice Problems

**Problem 1:** Choose your most complex function. Write a 1-minute explanation.

**Problem 2:** Practice explaining it out loud. How does it feel?

**Problem 3:** Ask a peer to listen. What questions do they ask?

<details>
<summary>💡 Hints</summary>

- Problem 1: Focus on "why" not "what"; discuss tradeoffs
- Problem 2: Practice builds confidence; aim for 2-3 run-throughs
- Problem 3: Questions help you clarify; they're good feedback

</details>
