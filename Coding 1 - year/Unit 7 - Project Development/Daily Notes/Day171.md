# Day 171: Writing Project Documentation

**Learning Target:** I can write clear, professional documentation including README and user guides.

---

## Key Concepts

### Documentation Types

**1. README (Project Overview)**
- What the program does
- How to run it
- Requirements
- Example usage

**2. User Guide (How to Use)**
- Step-by-step instructions
- Menus and options
- Examples with sample data

**3. Technical Documentation (How It Works)**
- Function descriptions
- Data structures
- File formats

---

## README Template

```markdown
# Grade Tracker

A Python program for students to track test scores, calculate averages, and generate grade reports.

## Features
- Add student names and scores
- Automatically calculate grade averages
- Assign letter grades (A-F)
- Save and load reports from files
- Formatted output for easy reading

## Requirements
- Python 3.6+
- No external libraries needed

## How to Run

1. Open a terminal/command prompt
2. Navigate to the project folder
3. Run: `python grade_tracker.py`

## Usage Example

```
===== GRADE TRACKER =====
1. Add student
2. View report
3. Exit

Enter choice: 1
Enter student name: Alice
Enter scores (type 'done' when finished):
Score 1: 85
Score 2: 92
Score 3: 78
Score 4: done

Grade Report:
- Student: Alice
- Scores: [85, 92, 78]
- Average: 85.0
- Grade: B
```

## File Format

Reports are saved as plain text files. Example:

```
=====================================
Grade Report: Alice
=====================================
Scores: [85, 92, 78]
Average: 85.0
Grade: B
=====================================
