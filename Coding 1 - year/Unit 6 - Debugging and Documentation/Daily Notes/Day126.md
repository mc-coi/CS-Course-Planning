# Day 126: Print Debugging

**Learning Target:** I can use print() statements strategically to debug programs.

### Key Concepts

**Print debugging** is one of the most powerful tools you have. The idea is simple: add print statements to trace what your program is doing at each step. By printing out variable values, you can see where things go wrong.

Good debugging print statements show:
- **What variables contain at key points** — their values at critical moments
- **Whether conditions are true or false** — so you know which branch executes
- **Which branch of an if/elif/else block is executing** — to trace control flow
- **Loop iteration counts and values** — to find off-by-one errors or missing data

The key is being strategic: print just enough to understand what's happening, but not so much that you're overwhelmed with output. Use a consistent format like `print(f"DEBUG: variable_name = {variable_name}")` so your debug output stands out from regular output.

### Code Examples

```python
# Example 1: Debugging a buggy loop
# Goal: Print multiples of 3 from 3 to 30
for i in range(1, 11):
    multiple = i * 3
    print(f"DEBUG: i = {i}, multiple = {multiple}")  # Strategic print
    if multiple > 15:
        print(multiple)

# Output shows us which multiples get printed and which don't

# Example 2: Debugging function issues
def calculate_discount(price, discount_percent):
    print(f"DEBUG: Entered function with price={price}, discount_percent={discount_percent}")
    discount_amount = price * (discount_percent / 100)
    print(f"DEBUG: discount_amount = {discount_amount}")
    final_price = price - discount_amount
    print(f"DEBUG: final_price = {final_price}")
    return final_price

result = calculate_discount(100, 20)
print(f"Final result: {result}")

# Example 3: Debugging a list problem
scores = [85, 90, 78, 92, 88]
total = 0
for score in scores:
    print(f"DEBUG: Adding {score}, total was {total}")
    total = total + score
    print(f"DEBUG: total is now {total}")
print(f"Final total: {total}")
```
