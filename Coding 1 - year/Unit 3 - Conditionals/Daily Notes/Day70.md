## Assessment Review

### Part 1: Written Questions Review

**Common Mistake Areas:**
1. Confusing `=` and `==`
   - Tip: Remember: assignment (=) vs comparison (==)

2. Mixing `and` and `or`
   - Tip: "Multiple values" usually means `or`

3. Forgetting about truthy/falsy
   - Tip: Practice: `bool(0)`, `bool("")`, `bool([])` all False

4. Order in elif chains
   - Tip: Always put most specific first

**Review Questions:**
1. Did you explain WHY, not just what?
2. Are your examples correct?
3. Did you use complete sentences?

---

### Part 2: Code Tracing Review

**If You Missed:**
1. Off-by-one operators (> vs >=)
   - Review comparison operator meanings

2. Short-circuit evaluation
   - `and` stops at first False
   - `or` stops at first True

3. Order of elif checking
   - Python stops at first True match

**Key Skill:**
Trace line-by-line, evaluating conditions step-by-step.

**Practice Method:**
Write out each condition's result before moving forward.

---

### Part 3: Code Writing Review

**Common Issues:**

**Issue 1: Missing validation**
- Check types: int vs string
- Check ranges: 0-100 for scores

**Issue 2: Wrong operator order**
- More specific conditions first in elif
- Proper and/or usage

**Issue 3: Incomplete feedback**
- Explain WHY someone can't do something
- List all missing requirements

**Improvement Strategies:**
- Plan before coding (flowchart/outline)
- Test with multiple inputs
- Add comments explaining logic
