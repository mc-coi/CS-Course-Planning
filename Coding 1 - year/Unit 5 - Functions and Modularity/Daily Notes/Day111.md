# Day 111: Top-Down Design

**Learning Target:** I can use top-down design to break large problems into smaller functions and solve them systematically.

### Key Concepts

**Top-down design** (also called **decomposition**) is a problem-solving strategy: Start with the big picture, then break it into smaller pieces that are easier to understand and solve.

Here's the approach:
1. **Understand the problem** — What is the program supposed to do?
2. **Identify major tasks** — What are the big steps?
3. **Break into functions** — Create a function for each major task
4. **Design function interactions** — How do functions call each other?
5. **Implement from bottom up** — Write helper functions first, then main functions

**Why top-down design?** Without it, code becomes a tangled mess ("spaghetti code") where everything depends on everything else. With it, you create a clear hierarchy where simple functions build on each other.

**Example:** Building a quiz game:
- Big task: "Run a quiz"
- Break down: "Load questions", "Ask each question", "Score the quiz", "Show results"
- Each becomes a function
- Main function calls them in order

The result is clear, testable, and easy to modify.

### Code Examples

```python
# Top-down design example: Quiz game

# Helper functions (low level)
def load_questions():
    """Load quiz questions from somewhere."""
    return [
        {"q": "What is 2+2?", "a": "4"},
        {"q": "What is the capital of France?", "a": "Paris"},
    ]

def ask_question(question_dict):
    """Ask one question and return True if correct."""
    print(f"Question: {question_dict['q']}")
    answer = input("Your answer: ")
    return answer.lower() == question_dict['a'].lower()

def score_quiz(score, total):
    """Calculate percentage and return grade."""
    percentage = (score / total) * 100
    if percentage >= 90:
        return "A"
    elif percentage >= 80:
        return "B"
    else:
        return "C"

# Higher-level function that uses helpers
def run_quiz():
    """Main function: orchestrate the entire quiz."""
    questions = load_questions()
    score = 0

    for q in questions:
        if ask_question(q):
            score = score + 1

    grade = score_quiz(score, len(questions))
    print(f"You got {score}/{len(questions)} correct!")
    print(f"Grade: {grade}")

# Entry point
if __name__ == "__main__":
    run_quiz()
```

This design is clean because:
- Each function has one job
- Functions don't know about each other unnecessarily
- The main function reads like a clear outline
- It's easy to test individual parts
