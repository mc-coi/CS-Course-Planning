# Day 102: The Global Keyword and Why to Avoid It

**Learning Target:** I understand the `global` keyword, why it exists, and why good design avoids it.

### Key Concepts

Yesterday, we learned that variables created inside a function are local and don't affect global variables. But what if you *want* a function to modify a global variable? That's where the **`global` keyword** comes in.

By writing `global variable_name` inside a function, you tell Python: "When I use this variable name, I'm referring to the global version, not a local one." Now if you assign to that variable, you're modifying the global one.

**Example:**
```python
score = 0

def add_points(points):
    global score
    score = score + points

add_points(10)
print(score)  # 10 (the global variable was modified!)
```

**However, good programmers avoid the `global` keyword.** Why? Because it creates **hidden dependencies**. When a function modifies global variables, it has **side effects**—it changes the state of the program in ways that aren't obvious from the function name. This makes code hard to debug and test. It also makes functions dependent on each other in unpredictable ways.

**Better design:** Instead of modifying global variables, have functions **return values**. The caller can then decide what to do with the result.

```python
# Bad (uses global):
score = 0
def add_points(points):
    global score
    score = score + points

# Good (uses return):
def add_points(current_score, points):
    return current_score + points

score = 0
score = add_points(score, 10)
```

The "good" version is clearer: the function takes input and produces output. It doesn't secretly modify global state.

### Code Examples

```python
# Example 1: Using global (works but not recommended)
counter = 0

def increment():
    global counter
    counter = counter + 1

increment()
increment()
print(counter)  # 2
```

```python
# Example 2: Better approach—return the new value
def increment(current_counter):
    return current_counter + 1

counter = 0
counter = increment(counter)
counter = increment(counter)
print(counter)  # 2
```

```python
# Example 3: Function composition with returns
def calculate_cost(quantity, price_per_item):
    return quantity * price_per_item

def apply_discount(cost, discount_percent):
    return cost * (1 - discount_percent / 100)

# Chain the functions
item_cost = calculate_cost(5, 10)  # 50
final_cost = apply_discount(item_cost, 10)  # 45
print(f"Final cost: ${final_cost}")
```
