### Sample Assessment Questions

(These are examples; your actual assessment may differ)

#### Part 1: Multiple Choice

1. Which of the following is correct function syntax?
   - a) `def function_name(parameters):`
   - b) `define function_name(parameters):`
   - c) `func function_name(parameters):`
   - d) `function function_name(parameters):`

2. What is the output?
   ```python
   x = 10
   def func():
       x = 5
       print(x)
   func()
   print(x)
   ```
   - a) 5, 5
   - b) 5, 10
   - c) 10, 5
   - d) Error

#### Part 2: Short Answer

3. Explain the difference between a parameter and an argument.

4. What does the `return` keyword do?

5. Why is it better to use `return` instead of `global` when you need to get a value out of a function?

#### Part 3: Coding

6. Write a function `is_vowel(letter)` that returns `True` if the letter is a vowel, `False` otherwise.

7. Write a function `count_vowels(text)` that counts and returns the number of vowels in a string.

8. Write a function `get_average(numbers)` that takes a list of numbers and returns the average.

9. Design and write a small program with 3+ functions that calculates the cost of a purchase with tax. Use the main() pattern.

10. Trace through this code and explain what it prints:
    ```python
    def double(x):

    ```

### Knowledge You'll Be Tested On

**Functions:**
- Correct syntax for defining a function
- How to call a function
- Parameters and arguments
- Return values
- When to use return vs. print

**Scope:**
- Local variables (created in functions)
- Global variables (created outside functions)
- What happens when you assign to a variable in a function
- When you can read a global variable from a function

**Composition:**
- Functions calling other functions
- Chaining function calls
- How data flows between functions

**Design:**
- Top-down design approach
- The main() function pattern
- How to break a problem into functions
- What makes good function design

**Best Practices:**
- Meaningful function names
- Appropriate use of parameters
- Using return values effectively
- Avoiding unnecessary global variables
- Writing docstrings
