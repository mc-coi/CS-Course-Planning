### Design Phase (Do This First!)

Before writing code, create a **design document**:

1. **Problem statement** — What does your program do?
2. **Inputs and outputs** — What does it take in? What does it produce?
3. **Major functions** — List all functions you'll write (name, purpose)
4. **Function interactions** — How do functions call each other?
5. **Data structures** — What lists/variables will you use?

**Example Design for a Quiz Game:**

```
=== Quiz Game Design ===

Problem: Create an educational quiz game

Inputs: Questions (list), User answers (strings)
Outputs: Score, Grade, Feedback

Functions:
1. load_questions() -> list
   - Returns a list of question dictionaries
   - Each has "question" and "answer" keys

2. display_question(q_dict) -> None
   - Prints question text nicely

3. get_answer() -> str
   - Gets user's answer from input

4. check_answer(user_ans, correct_ans) -> bool
   - Compares answers (case-insensitive)

5. calculate_score(correct, total) -> int
   - Returns percentage

6. get_grade(percentage) -> str
   - Returns A, B, C, D, or F

7. show_results(score, grade) -> None
   - Displays final report

8. main() -> None
   - Controls entire program flow

Main Flow:
- Load questions
- Loop through each question:
  - Display question
  - Get user answer
  - Check if correct
  - Accumulate score
- Calculate final score
- Show results
```

### Implementation Phase (Start Here)

1. Write your design document (even if brief)
2. Implement helper functions (the low-level ones)
3. Test each helper as you write it
4. Implement main() last
5. Do full program testing

**Start small:** Don't try to build everything at once. Implement the core functionality first.
