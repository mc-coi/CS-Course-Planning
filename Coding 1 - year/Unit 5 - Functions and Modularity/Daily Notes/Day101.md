# Day 101: Variable Scope — Local vs. Global

**Learning Target:** I can identify whether a variable is local or global and explain how scope affects where variables can be used.

### Key Concepts

Every variable exists in a specific **scope**—a region of code where the variable is accessible. Understanding scope prevents bugs and helps you write cleaner code.

A **local variable** is created inside a function and exists only inside that function. Once the function finishes, the local variable disappears. Each time you call the function, a new local variable is created. Local variables are the default in Python.

A **global variable** is created outside any function (at the top level of the file) and is accessible everywhere—in any function, anywhere in the program. Global variables exist from the moment they're defined until the program ends.

Here's a critical rule: **Inside a function, if you use a variable name, Python first looks for a local variable with that name. If it doesn't find one, it looks for a global variable.** This means you can *read* global variables from inside functions, but if you *assign* to a variable inside a function, you create a local variable (unless you use the `global` keyword—more on that tomorrow).

**Why does scope matter?** Functions are supposed to be independent. If functions randomly modified global variables, your program would be unpredictable and hard to debug. By using local variables, you keep functions isolated and safe.

### Code Examples

```python
# Example 1: Local vs. global variables
x = 100  # Global variable

def example_function():
    x = 5  # Local variable (same name, but different!)
    print(f"Inside function: x = {x}")

print(f"Before function: x = {x}")  # 100
example_function()  # Inside function: x = 5
print(f"After function: x = {x}")   # 100 (unchanged!)
```

```python
# Example 2: Reading a global variable from a function
name = "Alice"  # Global variable

def greet():
    # We can READ the global variable
    print(f"Hello, {name}!")

greet()  # Hello, Alice!
```

```python
# Example 3: Each function call gets its own local variables
def count_up(n):
    count = 0  # Local variable
    for i in range(n):
        count = count + 1
    print(f"Count: {count}")

count_up(3)  # Count: 3
count_up(5)  # Count: 5
# The 'count' variable was created fresh each time
```
