# Day 117: Review — Function Syntax, Parameters, Return Values

**Learning Target:** I can apply and explain the fundamentals: function definitions, parameters, arguments, and return values.

### Key Concepts to Review

#### Function Definition Syntax
```python
def function_name(parameters):
    # Function body
    return value
```

Key points:
- `def` keyword starts definition
- Name should be descriptive, lowercase with underscores
- Parameters go in parentheses (can be zero)
- Colon at the end
- Body must be indented
- `return` sends value back to caller

#### Parameters vs. Arguments
- **Parameter** — Variable in function definition
- **Argument** — Actual value passed when calling function

```python
def greet(name):  # 'name' is parameter
    print(f"Hello, {name}!")

greet("Alice")  # "Alice" is argument
```

#### Return Values
- Functions can return any type: int, string, list, bool, etc.
- `return` stops function immediately
- If no `return`, function returns `None`
- Returned value can be used, stored, or passed to another function

#### Function Composition
- Functions can call other functions
- Useful for breaking problems into pieces
- One function's return value becomes another's argument

### Common Mistakes to Avoid

1. **Forgetting the colon** after function definition
2. **Wrong indentation** in function body
3. **Confusing parameters and arguments** — parameters are in def line, arguments are in call
4. **Forgetting to return** — printing is not returning
5. **Not understanding call order** — code after a return doesn't execute
6. **Modifying a list without realizing** it's passed by reference

### Quick Reference

| Concept | Example |
|---------|---------|
| Define function | `def add(a, b):` |
| Call function | `result = add(5, 3)` |
| Return value | `return a + b` |
| Function with loop | `for x in items: total += x` |
| Function with conditional | `if x > 0: return True` |
| Function calls function | `square(double(5))` |
