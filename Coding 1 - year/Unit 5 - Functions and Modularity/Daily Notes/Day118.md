# Day 118: Review — Scope, Composition, and Modular Design

**Learning Target:** I can apply and explain advanced concepts: variable scope, function composition, and modular design patterns.

### Key Concepts to Review

#### Variable Scope

**Local variables:** Created inside a function, exist only in that function.
```python
def example():
    x = 5  # Local to example()
    print(x)  # Works

print(x)  # ERROR: x doesn't exist here
```

**Global variables:** Created outside any function, accessible everywhere.
```python
name = "Alice"  # Global

def greet():
    print(name)  # Can READ global

greet()  # Prints "Alice"
```

**Key rule:** If you assign to a variable inside a function, it's local (unless you use `global` keyword, which is rare).

#### Function Composition

Functions calling other functions:
```python
def double(n):
    return n * 2

def add_ten(n):
    return n + 10

def composed(n):
    return add_ten(double(n))  # compose them
```

Benefits:
- Break big problems into small pieces
- Reuse functions
- Clear, readable code

#### The main() Function Pattern

Professional structure:
```python
# Helper functions first
def helper_1():
    ...

def helper_2():
    ...

# Main function that orchestrates
def main():
    helper_1()
    helper_2()

# Call main at bottom
if __name__ == "__main__":
    main()
```

#### Top-Down Design

Approach:
1. Understand the big problem
2. Break into major tasks
3. Create a function for each task
4. Design how functions call each other
5. Implement bottom-up (helpers first)

### Common Mistakes to Avoid

1. **Forgetting that function creates new scope** — each function call gets fresh local variables
2. **Using global when you shouldn't** — use return values instead
3. **Modifying global variables inside functions** — leads to hidden dependencies
4. **Forgetting that assignment creates local variable** — `x = 5` inside function creates local, even if global x exists
5. **Not understanding call order** — which function runs first?

### Design Patterns to Know

**Pattern 1: Helper + Main**
```python
def helper():
    return ...

def main():
    result = helper()
    print(result)
```

**Pattern 2: Composition Chain**
```python
def f1(x): return x * 2
def f2(x): return x + 10
def f3(x): return f2(f1(x))
```

**Pattern 3: Loop in Function**
```python
def process_list(items):
    result = []
    for item in items:
        result.append(transform(item))
    return result
```

**Pattern 4: Conditional Return**
```python
def validate(value):
    if is_valid(value):
        return True
    else:
        return False
```

### Examples to Study

**Example 1: Well-Designed Program**
```python
def get_grades():
    # Input
    pass

def calculate_average(grades):
    # Processing
    pass

def display_report(avg):
    # Output
    pass

def main():
    grades = get_grades()
    avg = calculate_average(grades)
    display_report(avg)
```

Good design because:
- Each function does one thing
- No global variables
- main() is clear and simple
- Easy to test each part
