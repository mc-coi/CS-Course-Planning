# Day 115: Mini-Project Work Day — Modular Program of Your Choice

**Learning Target:** I can design and build a complete, modular program that solves a problem I care about.

### Overview

This is a **free-choice work day**. You have the entire period to build a modular program. You can:
- Continue and enhance the program from Days 113–114
- Start a completely new project of your choice
- Build something inspired by your interests

The key requirement: **Use what you've learned about modular design**. Your program should have:
- Multiple functions, each with a clear purpose
- Good documentation (docstrings and comments)
- Proper use of scope (local variables, return values)
- A clear main() function
- Thorough testing

### Requirements

Your program should:
1. **Solve a real problem** — It should do something useful or fun
2. **Use 5+ functions** — Each with a clear purpose
3. **Have good documentation** — Docstrings and comments
4. **Use proper scope** — Local variables, return values (no unnecessary globals)
5. **Have a main() function** — That orchestrates the program
6. **Be fully tested** — Work with multiple inputs without errors
7. **Be 200+ lines** — Substantial program showing your skills

### Design Process

1. **Decide on your idea** — What will your program do?
2. **Design functions** — List all functions you need
3. **Sketch the flow** — How does main() orchestrate them?
4. **Implement** — Write functions, test them
5. **Integrate** — Connect them in main()
6. **Test thoroughly** — Find and fix bugs
7. **Refine** — Improve code clarity and efficiency

### Code Structure

Use this template:

```python
"""
[Program Title]
[Brief description of what it does]
[Author and date]
"""

# ===== CONSTANTS & INITIAL DATA =====
# Any constants or data you need at startup

# ===== HELPER FUNCTIONS =====
# Low-level utility functions

def helper_1():
    """Brief description."""
    pass

def helper_2():
    """Brief description."""
    pass

# ===== PROCESSING FUNCTIONS =====
# Functions that do the main work

def process_1():
    """Brief description."""
    pass

# ===== MAIN PROGRAM =====
# The main function that controls flow

def main():
    """Main program flow."""
    # Your orchestration here
    pass

if __name__ == "__main__":
    main()
```

### Work Strategy

**If you have a clear idea:**
1. Design on paper first (5 min)
2. Write helper functions (15 min)
3. Test helpers (5 min)
4. Write main() (10 min)
5. Integrate and test (10 min)
6. Refine and document (5 min)

**If you're unsure:**
1. Look at the project ideas above
2. Pick the one that interests you most
3. Design it (what are the main tasks?)
4. Start coding the easy parts first
5. Build up from there

### Questions to Guide Design

- What does the user do first?
- What does the program need to do in response?
- What functions would make that happen?
- Can any of those functions be broken into smaller functions?
- What data flows between functions?

### Tips

- **Start simple** — Get basic functionality working, then add features
- **Test often** — Run your program frequently to catch bugs early
- **Use meaningful names** — `get_user_choice()` is better than `get_input()`
- **Avoid global variables** — Use return values and parameters instead
- **Document as you go** — Don't leave it for the end
- **Don't be afraid to start over** — Sometimes a fresh approach is better

### Getting Help

If you're stuck:
- **Review previous days' examples** — Look at patterns from Days 96–114
- **Break it into tiny pieces** — Get one small function working
- **Test in isolation** — Don't worry about main() until helpers work
- **Ask questions** — There are no dumb questions
