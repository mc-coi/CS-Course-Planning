# Day 104: Docstrings and Function Documentation

**Learning Target:** I can write clear docstrings that explain what a function does, what parameters it expects, and what it returns.

### Key Concepts

A **docstring** is documentation built into the function definition. It's a string (usually in triple quotes) that appears right after the `def` line and explains the function's purpose.

Docstrings are different from comments. Comments explain *how* code works; docstrings explain *what* a function does. Docstrings are especially important because other programmers (and future you!) will need to understand what your functions do without reading the entire implementation.

Good docstrings include:
- **What the function does** — a one-line summary
- **Parameters** — what each parameter represents and its type
- **Returns** — what the function returns and its type
- **Examples (optional)** — sample usage

Here's the basic format:
```python
def function_name(param1, param2):
    """
    One-line summary of what this function does.

    Parameters:
    param1 (type): Description of param1
    param2 (type): Description of param2

    Returns:
    type: Description of what is returned
    """
    # Implementation
```

When you use the help function in Python, it displays the docstring: `help(function_name)` or `print(function_name.__doc__)`.

### Code Examples

```python
# Example 1: Simple docstring
def greet(name):
    """
    Greets a person by name.

    Parameters:
    name (str): The person's name

    Returns:
    str: A greeting message
    """
    return f"Hello, {name}!"

print(greet("Alice"))
help(greet)
```

```python
# Example 2: Docstring for a math function
def calculate_area(radius):
    """
    Calculates the area of a circle given its radius.

    Parameters:
    radius (float): The radius of the circle in units

    Returns:
    float: The area of the circle in square units

    Example:
    >>> calculate_area(5)
    78.5
    """
    return 3.14159 * radius * radius

print(calculate_area(5))
help(calculate_area)
```

```python
# Example 3: Docstring for a function with multiple parameters
def create_account(username, email, age):
    """
    Creates a new user account with validation.

    Parameters:
    username (str): The desired username (2-20 characters)
    email (str): The user's email address
    age (int): The user's age

    Returns:
    bool: True if account created successfully, False if invalid input
    """
    if len(username) < 2 or len(username) > 20:
        return False
    if age < 13:
        return False
    return True
```

```python
# Problem 1 Example
def fahrenheit_to_celsius(f):
    """
    Converts a temperature from Fahrenheit to Celsius.

    Parameters:
    f (float): Temperature in Fahrenheit

    Returns:
    float: Temperature in Celsius

    Example:
    >>> fahrenheit_to_celsius(32)
    0.0
    >>> fahrenheit_to_celsius(212)
    100.0
    """
    return (f - 32) * 5/9
```

```python
# Problem 2 Example
def is_valid_password(password):
    """
    Checks if a password meets minimum security requirements.

    A valid password must be at least 8 characters long.

    Parameters:
    password (str): The password to validate

    Returns:
    bool: True if password is valid (8+ characters), False otherwise

    Examples:
    >>> is_valid_password("abc123")
    False
    >>> is_valid_password("secretpass")
    True
    """
    return len(password) >= 8
```

```python
# Problem 3 Example
def calculate_gpa(grades):
    """
    Calculates the GPA from a list of letter grades.

    Parameters:
    grades (list): List of single-character letter grades (A, B, C, D, F)

    Returns:
    float: The calculated GPA on a 4.0 scale

    Grade values: A=4.0, B=3.0, C=2.0, D=1.0, F=0.0

    Example:
    >>> calculate_gpa(['A', 'B', 'A', 'C'])
    3.25
    """
    grade_values = {'A': 4.0, 'B': 3.0, 'C': 2.0, 'D': 1.0, 'F': 0.0}
    total = sum(grade_values[grade] for grade in grades)
    return total / len(grades)
```
