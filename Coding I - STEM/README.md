# Coding I — Full Semester Course Materials

A complete, standards-aligned Python programming course for high school students. Built for an 18-week semester meeting 3 days per week (54 total class sessions), covering Python fundamentals from first program to capstone project.

## Getting Started — Installing Python

Before students can write code, they need Python and a code editor set up. Here's what to do depending on your devices.

### On a Windows Machine
1. **Install Python** — Free from the Microsoft Store. Search "Python" and install.
2. **Install Visual Studio Code** — Free from the Microsoft Store. Search "Visual Studio Code" and install.
3. **Install the Python Extension** — In VS Code, open Extensions (`Ctrl+Shift+X`), search "Python," install the official Microsoft extension.

### On a Chromebook (Also Works on Windows/Mac If You Can't Install)
Use **Kira Learning** — a free browser-based IDE with no installation required.
- Main site: [app.kira-learning.com](https://app.kira-learning.com)
- Direct IDE link: [app.kira-learning.com/ide/](https://app.kira-learning.com/ide/)

## Unit Descriptions

### Unit 1 — Introduction to Programming (Days 1–6)
Students are introduced to Python, the development environment, and computational thinking. Topics: `print()`, `input()`, variables, basic data types, and writing first programs. Ends with a mini-project applying all introductory concepts.

### Unit 2 — Variables, Data Types & Expressions (Days 7–15)
Deep dive into how Python stores and manipulates data. Topics: integers, floats, strings, booleans, type conversion, arithmetic operators, string methods, f-strings, and user input. Ends with an expression-based project.

### Unit 3 — Conditionals (Days 16–24)
Teaching programs to make decisions. Topics: `if`/`elif`/`else`, comparison operators, boolean logic (`and`/`or`/`not`), nested conditionals, and combining conditionals with input. Ends with a decision-tree style project.

### Unit 4 — Loops & Iteration (Days 25–33)
Making programs repeat. Topics: `for` loops with `range()`, iterating over lists and strings, `while` loops, `break`/`continue`, accumulator patterns, and nested loops. Ends with a loop-driven project.

### Unit 5 — Functions & Modularity (Days 34–42)
Writing reusable, modular code. Topics: defining and calling functions, parameters and arguments, return values, variable scope, default parameters, multiple return values, functions with lists, and modular program design. Ends with a modular application project.

### Unit 6 — Debugging & Documentation (Days 43–48)
Professional coding skills. Topics: error types (syntax/runtime/logic), debugging strategies, file I/O (`open`/`read`/`write`), exception handling (`try`/`except`/`finally`), raising exceptions, writing docstrings and comments. Ends with a data analyzer project.

### Unit 7 — Project Development (Days 49–54)
Capstone unit applying all skills. Topics: project planning, pseudocode, bottom-up development, unit and integration testing, code refactoring, polish and documentation, and student presentations. Ends with student-designed final projects and course reflection.

## File Types Explained

### Daily Lesson Plans (Word & PDF)
One file per class session. Each lesson plan follows a consistent format:
- **Objective** — what students will be able to do by the end
- **Warm-Up** — brief review or hook activity
- **Direct Instruction** — new concept with code examples
- **Guided Practice** — teacher-led coding activity
- **Independent Practice** — student coding exercises
- **Closure** — exit ticket or reflection


### Unit Assessments (Word and PDF)
Each unit includes a 20–25 question assessment in three formats:
- **.docx / .pdf** — printable version for traditional testing

Assessment question types include multiple choice, true/false, predict-the-output, write-the-code, identify-the-error, and short answer.

### Markdown Study Guides (.md)
One comprehensive study guide per day per unit. Each guide includes:
- Unit overview and learning targets
- Day-by-day key concepts with annotated code examples
- Practice problems at three tiers (Beginner / Intermediate / Challenge)
- Hint and answer collapsibles (`<details>` blocks)
- 15-problem practice bank
- Common mistakes table
- 10 assessment prep Q&A pairs
- Vocabulary and quick reference section

These are designed for student self-study, review, and reference. They render well in GitHub, VS Code, Jupyter, and most Markdown viewers.

## Standards Alignment

All materials are aligned to the **Tennessee Coding I** standards. The full mapping is in `TN_Coding_I_Full_Semester_Standards_Alignment.docx/.pdf` at the root of this folder. Key standard areas covered:

- Computational thinking and problem decomposition
- Variables, data types, and expressions
- Control structures (conditionals and loops)
- Functions and modularity
- Input/output and file handling
- Debugging and error handling
- Documentation and code quality
- Project planning and development process

## How to Use These Materials

**For daily instruction:** Open the `Word/Day N.docx` (or the matching PDF) for the current class session. Each lesson is self-contained and designed for a 60-minute period.

**For assessments:** Print the `.pdf` version from the `PDF/` folder.

**For student reference:** Share the unit's `.md` study guide — it's the go-to resource for review, homework help, and test prep.

**For sub plans:** The PDF lesson plans are fully self-contained. A substitute can run any lesson directly from the PDF with no additional context.
