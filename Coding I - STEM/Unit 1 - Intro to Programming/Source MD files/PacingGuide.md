# Unit 1 - Introduction to Programming: Pacing & Differentiation Guide

## Unit Overview
- **Total days allocated:** 6 days (compressed for STEM semester)
- **Core topics covered:** Computer science fundamentals, algorithms, Python syntax (print, variables, data types, user input, basic arithmetic), 5-step problem-solving process
- **Prerequisites students need:** Comfort with basic mathematics; computer access with Python/IDE; ability to follow procedural instructions

---

## Week-by-Week Pacing

### Week 1 (Days 1–3): Fundamentals & First Programs

#### Day 1: What is Computer Science?
- **Essential days:** YES—foundational concept
- **Key learning:** Algorithm definition, flowcharts, computer science as problem-solving
- **What to teach:** Algorithm characteristics (precise, unambiguous, finite, effective); flowchart symbols; connection between problems and algorithms
- **If ahead of pace:** Have students design flowcharts for more complex everyday tasks (cooking, morning routine); introduce pseudocode concept early
- **If behind pace:** Focus only on print statements and basic algorithm definition; skip detailed flowchart practice for now

#### Day 2: Setting Up Python & First Program
- **Essential days:** YES—students must write and run code
- **Key learning:** IDE setup, print() function, comments, basic syntax
- **What to teach:** Installation or using online IDE; writing first program; syntax rules for print; comment purpose
- **If ahead of pace:** Have students create ASCII art or multi-line creative outputs; introduce string concatenation with commas
- **If behind pace:** Use online IDE (Kira Learning) exclusively; skip local installation; focus on print() only

#### Day 3: Variables & Data Types
- **Essential days:** YES—core programming concept
- **Key learning:** Variables as containers, four data types (str, int, float, bool), type() function
- **What to teach:** Variable naming rules; differentiating between data types; why types matter; reassignment
- **If ahead of pace:** Introduce f-strings early; practice type conversion; challenge students with type guessing puzzles
- **If behind pace:** Focus on str and int only; skip bool for now; use simple variable examples; skip type() function—save for review

---

### Week 2 (Days 4–6): User Input & Problem-Solving

#### Day 4: User Input & Simple Calculations
- **Essential days:** YES—interactive programs require this
- **Key learning:** input() function always returns strings; type conversion; arithmetic operators; proper use of int() and float()
- **What to teach:** input() mechanics; why conversion is needed; all seven arithmetic operators; practical calculator examples
- **If ahead of pace:** Introduce modulus (%) with real-world examples (remainders, schedules); explore order of operations; build more complex calculators
- **If behind pace:** Focus on input(), int(), float(), and +/- operators only; skip //, %, ** for now; use simpler problems

#### Day 5: Problem Solving with Python
- **Essential days:** YES—students need to practice the process
- **Key learning:** 5-step software development process (Define, Plan, Code, Test, Refine); applying it to real problems
- **What to teach:** Each step with examples; pseudocode concept; testing different inputs including edge cases; how to improve code
- **If ahead of pace:** Have students optimize code (fewer lines, better variable names); challenge with edge cases; introduce debugging mindset
- **If behind pace:** Walk through one full example together (temperature converter); provide partially completed code as starter; reduce number of test cases required

#### Day 6: Unit 1 Project Day
- **Essential days:** FLEXIBLE (but strongly recommended)—integrates all unit concepts
- **Key learning:** Complete program synthesis; putting all skills together
- **What to teach:** Review the project requirements; allow time for troubleshooting and testing
- **If ahead of pace:** Have students add features (error handling, input validation, more calculations); require use of f-strings; bonus: add more complex formulas
- **If behind pace:** Provide more scaffolding: starter code with skeleton structure; checklist of what to include; allow simpler program with fewer requirements

## Differentiation Strategies

### For Struggling Students

**Scaffolding tips for this unit:**
- Break Day 2 setup into smaller steps with screenshots/video guides
- Provide code templates with blanks to fill in
- Use only the online IDE (Kira Learning) to reduce setup friction
- Start with simple print() statements before variables
- Use int() and float() before exploring other operators
- Provide worked examples before each practice problem

**Which practice problems to assign first (easiest wins):**
- Problems 1 from each day (Beginner tier)
- Focus on Unit 1: Practice Bank items 1–3 first
- Have students copy-and-run provided code before writing their own
- Build confidence with print() and simple variables before input()

**Suggested pacing adjustments:**
- Spread Unit 1 over 2.5 weeks if possible (8–9 days instead of 6)
- Spend 2 days on variables & data types instead of 1
- Combine Days 4–5 into problem-solving for calculators only
- Move project to Week 3 as a standalone day
- Skip flowcharts on Day 1; teach as part of Day 5 planning step

**What prerequisite gaps to look for:**
- Weak basic math skills (order of operations, fractions, decimals)
- Typing/keyboard skills—slow input speed
- Abstract thinking about variables as "boxes"—use concrete analogies (Post-it notes, mailboxes)
- Comfort with error messages—normalize errors as learning

---

### For On-Track Students

**Standard path through the unit:**
- Follow the 6-day schedule as written
- Assign one Beginner + one Intermediate problem per day from the practice bank
- Complete the project on Day 6 with standard requirements
- Encourage use of f-strings for formatting output

**Which practice problems to assign:**
- Days 1–3: Beginner + Intermediate problems
- Days 4–5: Intermediate + one Challenge problem
- Day 6: Full project with all requirements met

---

### For Advanced Students

**Extension activities specific to this unit's topics:**
- Require f-string formatting from Day 2 onward
- Explore all arithmetic operators (%, //, **) with real-world applications
- Design algorithms for non-programming problems (sorting, game rules, recipes)
- Create multi-step flowcharts with complex decision logic
- Introduce string methods: .upper(), .lower(), .replace() (preview)

**Challenge problems to assign:**
- Unit 1 Practice Bank items 11–15 (all Challenge tier)
- Day 4 Problem 3: Rectangle calculator with diagonal using math.sqrt()
- Day 5 Problem 3: Multi-currency converter with 3+ currencies
- Extra: Write a complete program simulating a real-world calculator (mortgage, compound interest, etc.)

**Ways to deepen understanding beyond the curriculum:**
- Ask: "What if the user enters invalid input? How would you handle that?"
- Explore variable scope (preview for later units)
- Have them write README files explaining their programs
- Challenge: Build a program that asks multiple questions and does multiple calculations (combine concepts)

## Flexibility Notes

### Natural pause points for re-teaching:
- **End of Day 2:** Before introducing variables (foundation: input/output works)
- **End of Day 3:** Before introducing calculations (foundation: data storage works)
- **End of Day 5:** Before project (foundation: problem-solving process is clear)

### Topics students most commonly struggle with:
- **Type conversion:** Why `int(input())` is needed; why `"5" + 3` fails
- **Variable reassignment:** Understanding that `x = 10` changes x; confusion with `x = x + 1`
- **Arithmetic with strings:** Cannot do math on strings; `+` concatenates strings, not adds numbers
- **Input always returns strings:** Students often forget to convert before using in calculations
- **Scope of variables:** Introducing them too early; keep simple "one script" focus

### Topics students typically move through quickly:
- **Basic print():** Most students pick this up immediately
- **Simple arithmetic:** Students with math background move fast through +, -, *, /
- **Comments:** Easy to understand; doesn't require practice
- **Boolean concept:** If properly explained with True/False, students get it quickly

## Suggested Checkpoints

### Quick formative checks to use mid-unit:

**After Day 2:**
- Can each student write and run a print() statement?
- Are all students set up with a working IDE?
- Checkpoint: "Print your name, grade, and one hobby on separate lines"

**After Day 3:**
- Can students create variables and print them?
- Do they understand type vs. value?
- Checkpoint: "Create four variables (one of each type) and print all of them"

**After Day 4:**
- Can students use input() and convert to numbers?
- Can they perform basic arithmetic?
- Checkpoint: "Ask for two numbers and print their sum, difference, product"

**After Day 5:**
- Can students articulate the 5-step process?
- Can they plan before coding?
- Checkpoint: "Describe the steps to convert miles to kilometers (no code yet)"

**Before Day 6:**
- Can students combine all concepts (variables, input, calculations, output)?
- Checkpoint: Quick 2-minute quiz on data types and arithmetic operators

## STEM-Specific Pacing Notes

This is a **semester-long course with fewer total days than full-year Coding I**. Unit 1 is compressed into 6 days instead of the full-year version's 7–8 days.

**Key implications:**
- Every day must count; very limited time for re-teaching mid-unit
- Students who fall behind by Day 3 will struggle in Units 2–7
- **Daily homework is essential** to reinforce concepts (even 15 min/day)
- Use synchronous class time efficiently: concept introduction + guided practice only
- Move problem-solving and project work partially to homework for on-track/advanced students
- Consider a "Pre-Unit 1 bootcamp" (1–2 days before official start) for students with low comfort with technology

**Recommended daily structure (if 50-min class periods):**
- 5 min: Hook/review
- 10 min: Concept instruction
- 20 min: Guided practice (live coding with students)
- 10 min: Assign homework + troubleshoot setup issues
- 5 min: Checkpoint question (exit ticket)

---

## Assessment & Mastery Targets

Students must demonstrate mastery of these by end of Unit 1:
1. Write a complete Python program with variables, input, and calculations
2. Explain the difference between data types and when to use each
3. Apply the 5-step problem-solving process to a new problem
4. Debug basic syntax errors (missing quotes, wrong capitalization, wrong type)
