# Unit 7 - Project Development: Pacing & Differentiation Guide

## Unit Overview
- **Total days allocated:** 6 days (capstone project for STEM semester)
- **Core topics covered:** Project planning and specification, breaking problems into functions, incremental development, systematic testing and debugging, complete documentation, professional presentation
- **Prerequisites students need:** Mastery of all Units 1-6; ability to work independently on extended projects; problem decomposition skills

---

## Week-by-Week Pacing (Final Capstone)

### Week 1 (Days 1–3): Planning & Design

#### Day 1: Project Planning & Design
- **Essential days:** YES—foundation for entire project
- **Key learning:** Problem definition, requirements specification, function design, pseudocode, test case planning
- **What to teach:** Software development process; problem statement clarity; breaking into functions; designing before coding; what makes a project "done"
- **If ahead of pace:** Complex project planning; multiple design iterations; formal specification documents
- **If behind pace:** Provide project template; partially filled specification; suggested functions

#### Day 2: Proposal & Specification
- **Essential days:** YES—define what you're building
- **Key learning:** Writing clear problem statement, identifying inputs/outputs, listing functions with descriptions, planning data structures, defining test cases
- **What to teach:** Making specifications testable; completeness checklist; predicting challenges; realistic scope
- **If ahead of pace:** Peer review of specifications; refining based on feedback
- **If behind pace:** Use provided template; fill in blanks; guided questions

#### Day 3: Implementation Begins
- **Essential days:** YES—moving from design to code
- **Key learning:** Starting with simplest functions, testing incrementally, building complexity gradually, avoiding big-bang integration
- **What to teach:** Building one function at a time, testing immediately, using helper functions, keeping code organized
- **If ahead of pace:** Parallel development of multiple functions; advanced debugging techniques
- **If behind pace:** Implement one complete function per session; provide starter code

---

### Week 2 (Days 4–6): Development & Presentation

#### Day 4: Continued Development & Testing
- **Essential days:** YES—getting project working
- **Key learning:** Incremental development, comprehensive testing, handling edge cases, debugging systematically
- **What to teach:** Testing strategy; edge cases and error conditions; documenting test results; iterative refinement
- **If ahead of pace:** Advanced testing; optimization; refactoring for clarity
- **If behind pace:** Simpler features; basic testing; focus on core functionality

#### Day 5: Documentation & Code Cleanup
- **Essential days:** YES—making code professional and understandable
- **Key learning:** Complete docstrings, inline comments, README file, explaining design choices, code organization
- **What to teach:** Documentation standards; what future readers need to know; making code readable; final review checklist
- **If ahead of pace:** Professional-grade documentation; API documentation
- **If behind pace:** Template docstrings; comments provided as starters; simple README

#### Day 6: Project Presentation & Reflection
- **Essential days:** YES—demonstrating learning and communicating results
- **Key learning:** Explaining code and design, demonstrating functionality, reflecting on learning, celebrating achievement
- **What to teach:** Presentation skills; explaining design choices; live demo; answering questions; personal reflection on growth
- **If ahead of pace:** Advanced presentations; peer feedback sessions; refinement based on feedback
- **If behind pace:** Guided presentation template; practice with smaller audiences first

---

## Project Scope & Requirements

### Minimum Requirements (All Students)
- **Problem statement:** Clear description of what program does
- **Code functionality:** Meets 100% of specifications (or documented exceptions)
- **Functions:** At least 5 well-designed functions
- **Data handling:** Appropriate data structures and I/O (files, user input)
- **Error handling:** Graceful handling of errors and invalid input
- **Testing:** Evidence of testing with multiple test cases
- **Documentation:** Docstrings for all functions, comments for complex logic, README
- **Presentation:** Live demo and explanation of design

### Enhanced Requirements (On-Track & Advanced Students)
- **Features:** Exceed basic requirements; add interesting features
- **Code quality:** Professional organization, clear naming, DRY principle
- **Testing:** Comprehensive test suite, edge cases covered
- **Optimization:** Efficient algorithms and runtime
- **Presentation:** Polished demo, insightful design explanations, peer evaluation

### Challenge Requirements (Advanced Students)
- **Complexity:** Sophisticated algorithms or data structures
- **Scale:** Handles larger datasets or real-world data
- **Innovation:** Novel features or approaches
- **Documentation:** Professional-grade documentation and README
- **Presentation:** Teaches others, compares approaches, reflects on tradeoffs

## Differentiation Strategies

### For Struggling Students

**Scaffolding tips for this unit:**
- Provide project options with varying difficulty (choose easiest)
- Give detailed specification with filled-in functions list
- Provide starter code with function signatures and comments
- Break Days 1-3 into smaller micro-deadlines (each function drafted)
- Offer check-ins after each function completion
- Provide debugging support during development

**Which project types to suggest:**
- Data analysis (read file, compute statistics, display results)
- Simple game (guess number, rock-paper-scissors with scoring)
- Calculator with special features (unit converter, grade calculator)
- Text processor (word counter, simple cipher)
- Data tracker (expense tracker, grade tracker)

**Suggested pacing adjustments:**
- Allow 1 extra day (7 days total) if possible
- Focus on functionality first; documentation can be added later
- Reduce feature count; aim for fewer functions working well
- Simplify or provide test cases
- Use pair programming or small group work
- Provide function implementation hints

**What prerequisite gaps to look for:**
- Weak function design (units 1-5 not mastered)
- Cannot plan before coding (jumps into code)
- Struggles with file I/O
- Doesn't test incrementally
- Difficulty with systematic debugging

---

### For On-Track Students

**Standard project path:**
- Choose from provided project options (moderate difficulty)
- Follow 6-day timeline exactly
- Complete all minimum requirements
- Aim for polished, working project
- Provide clear documentation and demo

**Project suggestions:**
- Grade tracker (input scores, compute stats, save to file)
- Quiz game (read questions from file, score, display results)
- Budget planner (input expenses, categorize, produce report)
- Text analysis (word frequency, readability, pattern finding)
- Data visualization (read data, compute and display statistics)

**Checkpoint expectations:**
- Day 1: Specification finalized and approved
- Day 2: Detailed function list and pseudocode complete
- Day 3: 2-3 functions implemented and tested
- Day 4: All functions implemented, basic integration working
- Day 5: Full testing complete, documentation drafted
- Day 6: Final review and presentation ready

---

### For Advanced Students

**Challenge project path:**
- Design own project or choose from advanced options
- Tackle sophisticated problem with multiple design decisions
- Implement with professional code quality
- Comprehensive testing including edge cases
- Detailed documentation and design explanation

**Project suggestions:**
- Build a small application with GUI (tkinter)
- Statistical analysis tool for real datasets
- Game with multiple levels/features (2048, Hangman, Sudoku solver)
- Data processing pipeline (ETL - Extract, Transform, Load)
- Encryption/compression utility
- Network tool or data scraper
- Scientific simulation or modeling

**Advanced expectations:**
- Multiple advanced functions with complex logic
- Sophisticated data structures (lists, dictionaries, potentially classes)
- Comprehensive error handling and validation
- Performance optimization
- Professional documentation with design rationale
- Peer code review and refactoring
- Presentation comparing approaches or discussing tradeoffs

## Project Evaluation Rubric

### Code Functionality (40%)
- [ ] Program meets all specifications
- [ ] Functions work correctly individually and together
- [ ] Handles normal and edge cases
- [ ] Error handling is graceful (no crashes)
- [ ] Input validation prevents invalid data

### Code Quality (20%)
- [ ] Clear, descriptive function and variable names
- [ ] Appropriate data structures for the problem
- [ ] Follows DRY principle (no significant duplication)
- [ ] Efficient algorithms
- [ ] Proper use of functions and modularity

### Documentation (20%)
- [ ] All functions have docstrings
- [ ] Complex logic has inline comments
- [ ] README explains what the program does
- [ ] Design choices are explained
- [ ] Code is easy to understand

### Testing & Debugging (10%)
- [ ] Evidence of systematic testing
- [ ] Multiple test cases documented
- [ ] Edge cases considered
- [ ] No remaining obvious bugs

### Presentation (10%)
- [ ] Clear explanation of problem and solution
- [ ] Live demonstration of functionality
- [ ] Can answer questions about design
- [ ] Presents professionally and confidently

## STEM-Specific Capstone Guidance

Unit 7 is the **culminating experience** of the course. For STEM students, the project should ideally apply programming to a scientific or mathematical problem.

### Recommended STEM Project Types:

**Physics/Engineering:**
- Projectile motion simulator
- Simple physics engine (gravity, collisions)
- Structure calculator (load, stress analysis)
- Pendulum simulator

**Biology/Chemistry:**
- Population dynamics simulation
- Enzyme kinetics calculator
- Genetic simulation
- Chemical reaction calculator

**Data Science:**
- Analyze experimental results
- Statistical analysis tool
- Data visualization
- Trend analysis

**Mathematics:**
- Fractal generator (Mandelbrot set)
- Prime number finder
- Mathematical sequence explorer
- Equation solver

**Environmental:**
- Carbon footprint calculator
- Weather data analyzer
- Resource usage simulator
- Pollution impact calculator

### STEM Success Criteria:
1. **Problem rooted in science/math:** Problem is from a STEM domain
2. **Accurate implementation:** Formulas/concepts are correctly applied
3. **Handles real data:** Works with realistic datasets or parameters
4. **Clear documentation:** Anyone could understand the science and code
5. **Generalizable:** Could be reused for similar problems
6. **Testable against known results:** Can verify correctness

## Final Reflection Prompts

After completing Unit 7, students should reflect on:

1. **Problem Solving:** How did breaking the problem into functions help?
2. **Programming Growth:** What's different about your coding now vs. Unit 1?
3. **Debugging:** What was hardest to debug and how did you solve it?
4. **Design:** Would you design this differently if you started over? Why?
5. **Testing:** What edge cases did you miss initially?
6. **Documentation:** Why is documentation important for your project?
7. **Challenge:** What feature would you add next if you had more time?
8. **Overall:** What are you most proud of? What would you improve?

## STEM Course Completion

Upon completion of Unit 7, students should be able to:
1. **Plan** a programming solution from problem statement to specification
2. **Design** programs using modular functions and appropriate data structures
3. **Implement** working solutions with professional code quality
4. **Test** systematically including edge cases and error conditions
5. **Debug** complex programs using systematic approaches
6. **Document** code clearly for others to understand and extend
7. **Present** their work explaining design choices and demonstrating functionality
8. **Reflect** on their growth and learning throughout the course

These are the foundations of computer science thinking—applicable far beyond Python!
