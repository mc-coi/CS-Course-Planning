# Unit 1: Unity Interface & Game Design Thinking

> Get comfortable in the Unity editor and start thinking like a game designer — understanding the language, tools, and mindset that separate great games from forgettable ones.

## Unit Overview

- **Duration:** 18 days / ~3.5 weeks
- **TN Standard:** 9-12.VGD.1 — Navigate the Unity development environment and apply game design principles

---

## Learning Objectives

By the end of this unit, you will:

1. **Game Design Foundation**
   - Understand the MDA framework (Mechanics, Dynamics, Aesthetics)
   - Analyze games using design patterns
   - Think critically about how systems create player experiences

2. **Unity Editor Proficiency**
   - Navigate the Scene, Game, Inspector, Hierarchy, and Project panels
   - Use keyboard shortcuts and Gizmos for efficient scene building
   - Understand the component-based architecture

3. **Scene Building Skills**
   - Create and customize GameObjects
   - Apply materials, colors, and lighting
   - Organize scenes with parenting, layers, and hierarchies

4. **Prefab System & Scripting Basics**
   - Create and reuse prefabs
   - Write basic C# scripts with Start() and Update()
   - Use Debug.Log() for debugging

5. **Play Mode Understanding**
   - Distinguish between Play Mode and Edit Mode
   - Understand what persists and what reverts
   - Build complete scenes ready for gameplay

## Key Vocabulary

- **GameObject** — The fundamental building block of a scene
- **Component** — A reusable module (Transform, Renderer, Collider, Script)
- **Prefab** — A reusable template for GameObjects
- **Material** — Defines how a surface looks (color, shininess)
- **Gizmo** — Visual tool for moving, rotating, scaling in the Scene View
- **Hierarchy** — Tree view of GameObjects in a scene (parent-child relationships)
- **MDA** — Mechanics (what players can do) → Dynamics (what happens) → Aesthetics (how it feels)

## Tips for Success

- **Keyboard shortcuts save time** — Learn W, E, R (move, rotate, scale) and F (frame)
- **Save frequently** — File → Save Scene (Cmd/Ctrl+S) is your friend
- **Use meaningful names** — "Wall_North" is better than "Cube1"
- **Organize with parenting and layers** — Saves sanity in complex scenes
- **Play Mode is safe** — Experiment freely; changes revert when you stop
- **Prefabs are your friend** — Reusable templates for consistency and efficiency
