# Day 1: Introduction to Game Design Theory & the MDA Framework

**Learning Target:** I can define the MDA framework (Mechanics, Dynamics, Aesthetics) and apply it to analyze real games.

---

## Key Concepts

**Mechanics** — The rules and systems that define what players *can* do (movement speed, jump height, health points, resource costs).

**Dynamics** — What happens when players *interact* with those mechanics (a slow jump feels floaty; a slow resource gain feels grindy; variable rewards feel rewarding).

**Aesthetics** — The emotional response or mood the game creates (fun, suspense, satisfaction, frustration, relaxation).

**MDA Framework** — A design thinking tool that connects Mechanics → Dynamics → Aesthetics. Design *down* from vision (what feeling do we want?), implement *up* (what mechanics create that feeling?).

**Game Loop** — The cycle of player action → game response → new state → player decision.

---

## Game Examples

| Game | Mechanics | Dynamics | Aesthetics |
|------|-----------|----------|-----------|
| **Tetris** | Pieces fall; rotate/move pieces; clear lines | Quick decision-making; pressure increases; satisfaction from perfect clears | Engaging, rhythmic, slightly stressful |
| **Portal** | Place portals; move through portals; momentum preserved | Spatial reasoning; experimentation; discovering solutions | Clever, satisfying, mentally rewarding |
| **Stardew Valley** | Plant crops; fish; socialize; manage stamina | Daily routine; slow progression; organic discovery | Relaxing, meditative, nostalgic |

---

## Guided Practice

1. **Choose a game you play regularly** (Minecraft, Fortnite, Candy Crush, Mario Kart — anything counts).
2. **List 3 core mechanics** — What are the main things players do?
3. **Describe the dynamics** — What happens *because* of those mechanics? How do they interact?
4. **Identify the aesthetics** — What mood or feeling does the game create?
5. **Apply MDA thinking** — If the designer wanted to make the game *more* exciting, which mechanic would they change? Why?

---

## Practice Problems

**Beginner:** Play for 5 minutes and list the top 3 mechanics you encountered.

**Intermediate:** Draw a simple diagram showing how one mechanic in your game creates a dynamic that leads to a specific aesthetic feeling.

**Challenge:** Propose ONE mechanic change to your game. Use MDA thinking to predict: How would it change the dynamics? Would the new dynamic support the same aesthetic, or a different one? Is that intentional?

<details>
<summary>💡 Hints</summary>
- Mechanics are things. Dynamics are what happens when those things interact with the player.
- One mechanic can create *multiple* dynamics depending on context.
- Aesthetics are how the player *feels*, not how the game *looks* (though visuals help).
</details>

<details>
<summary>✅ Sample Analysis (Minecraft)</summary>

**Mechanics:** Place blocks, dig blocks, craft items, health system

**Dynamics:** Exploration rewards creativity; resource scarcity creates planning; night cycles create time pressure; crafting gates progression

**Aesthetics:** Empowering (you build what you want), explorative, survival-driven in hard mode; relaxing in creative mode

**Change idea:** If we made crafting instant instead of requiring a crafting table, the dynamic changes from "planning expeditions to find resources" to "constant upgrading." The aesthetic shifts from exploratory to incremental/goal-focused. That's a trade-off.

</details>
