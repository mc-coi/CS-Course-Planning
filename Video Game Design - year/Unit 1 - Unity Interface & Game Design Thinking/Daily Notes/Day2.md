# Day 2: Famous Games & Design Patterns

**Learning Target:** I can identify design patterns in famous games and explain how mechanics create specific player experiences.

---

## Key Concepts

**Design Pattern** — A reusable solution to a common game design problem (progression gates, resource management, skill expression, feedback loops).

**Feedback Loop** — The cycle of player action → immediate response → motivation to act again. **Positive loops** accelerate engagement; **negative loops** create challenge and tension.

**Progression Curve** — How difficulty/reward changes over time. Good curves keep players in a "flow state" (not bored, not overwhelmed).

**Skill Expression** — Allowing players to show mastery and make meaningful decisions within the rules.

---

## Famous Game Design Patterns

### **Dark Souls / Elden Ring**
- **Pattern:** Punishing but fair difficulty; bonfire (checkpoint) system
- **Mechanic:** Stamina management, attack commitment
- **Dynamic:** Cautious, deliberate gameplay; learning through failure
- **Why it works:** Tight feedback (every action has weight); mastery feels earned

### **Candy Crush**
- **Pattern:** Variable reward schedules (sometimes you win big, sometimes small)
- **Mechanic:** Limited moves; cascading matches; power-ups
- **Dynamic:** Excitement + addiction through uncertainty
- **Why it works:** Psychological reward loop; always one more try

### **Super Mario Bros**
- **Pattern:** Gradual skill introduction; level design teaches mechanics
- **Mechanic:** Jump mechanics; enemy patterns; block surprises
- **Dynamic:** Intuitive mastery; each level escalates slightly
- **Why it works:** Players naturally learn by playing

---

## Guided Practice

1. **Pick a game famous for one mechanic** (jumping, building, puzzle-solving, exploration).
2. **Play or watch 10 minutes** and take notes on:
   - What's the first thing players learn?
   - How does difficulty escalate?
   - When do players feel rewarded?
3. **Sketch the feedback loop:** What encourages the player to keep playing?
4. **Identify the design pattern:** What problem is the designer solving?

---

## Practice Problems

**Beginner:** Name three games and one mechanic each that makes them unique.

**Intermediate:** Choose one game. Describe its progression curve: Is it linear (steady increase)? Does it plateau? Does it have difficulty spikes? Is that intentional?

**Challenge:** Redesign the feedback loop. If a game felt repetitive, what ONE mechanic change could re-engage players? Why would that work?

<summary>💡 Hints</summary>
- Look for moments where the game *tells* you you're doing well (visual feedback, sound, score, dialogue).
- Difficulty should increase *gradually* or the feedback loop breaks.
- Repetition is okay if the feedback feels fresh or meaningful.

<details>
<summary>✅ Sample Pattern Analysis (Portal)</summary>

**Design Pattern:** Mechanical literacy (player learns one rule, then breaks/extends it)

**Progression:** Test chambers 1-5 teach portals; chambers 6-12 add momentum; 13-19 require chaining mechanics; final chambers demand precision

**Feedback Loop:** Player succeeds → confidence → complexity → struggle → aha! moment → pride

**Why it works:** Players feel *smart*, not lucky. The game trusts their intelligence.

</details>
