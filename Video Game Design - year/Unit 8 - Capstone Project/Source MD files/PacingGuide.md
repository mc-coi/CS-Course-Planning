# Unit 8 - Capstone Project: Pacing & Differentiation Guide

## Unit Overview

- **Total days allocated:** 20 days (~4 weeks)
- **Course type:** Year-long traditional
- **Core topics covered:** Design document, scope, Core mechanics, features, polish, Bug fixes, playtesting, Demo, documentation
- **Prerequisites students need:** 
  - Comfort with basic computer navigation and file management
  - Understanding of game design vocabulary (from Unit 1 or prior knowledge)
  - Familiarity with C# basics (for units beyond Unit 1)

## Week-by-Week Pacing

(Days 1-3)

- **Essential days** (must teach): All days in this section form the critical path for understanding planning
- **Flexible days** (can compress if pressed for time): 
  - For year-long: Consider pairing planning practice with the next section
  - For STEM: Skip advanced exploration; stick to core competencies only
- **If ahead of pace:** 
  - Deep dive: Have students implement additional features or variations
  - Exploration: Encourage experimentation with advanced properties and parameters
  - Refactoring: Ask students to improve or extend their solutions
- **If behind pace:** 
  - Focus on core competencies first
  - Use guided practice heavily before independent practice

## Differentiation Strategies

### For Struggling Students

**Scaffolding & Support:**
- Provide starter templates for all scripts and scenes
- Pair with a stronger student or provide additional office hours
- Use the Reference Guide heavily; mark key sections for quick lookup
- Break complex problems into smaller sub-problems
- Offer alternative, simpler versions of practice problems

**Which practice problems to assign first:**
- Start with Beginner/Tier 1 problems only
- Ensure each concept is mastered before moving to Intermediate
- Use worked examples before asking for independent coding

**Pacing adjustments:**
- Add an extra day to each section if possible
- Extend guided practice portions and reduce independent practice
- Allow use of notes and reference materials on assessments


### For On-Track Students

**Standard path through the unit:**
- Follow the day-by-day structure as outlined above
- Complete all Intermediate tier practice problems
- Mini-project should include all core features plus one small enhancement

**Which practice problems to assign:**
- Complete Beginner and Intermediate tiers
- For additional challenge, attempt Challenge tier problems
- Use practice problems as formative assessment checkpoints

### For Advanced Students

**Extension activities specific to this unit:**
- Implement additional features not covered in the base unit
- Optimize code or scenes for performance
- Create variations or higher-difficulty versions of practice problems
- Mentor peers or create tutorials for other students

**Challenge problems or features to add:**
- Challenge tier problems from practice bank
- Design and implement original mechanics using unit concepts
- Integrate concepts from multiple units into complex projects
- Research and present advanced Unity features related to the unit

**Ways to deepen understanding:**
- Read official Unity documentation on unit topics
- Analyze how published games use these concepts
- Create educational content (tutorials, guides, video explanations)
- Help other students debug complex problems

## Flexibility Notes

### Natural pause points for re-teaching
- Between major concept sections (ideal time for review quizzes or mini-assessments)
- After any section where struggling students show gaps
- Before moving to dependent concepts (e.g., input review before controls unit)

### Topics students commonly struggle with
- **C# and scripting:** Getting GetComponent() to work, understanding scope and variable lifetime
- **Physics:** The difference between forces and velocity; when to use different Rigidbody settings
- **UI:** Canvas scaling and positioning across different resolutions
- **Hierarchy and parenting:** World vs local space; scale inheritance causing unexpected results

### Topics students typically move through quickly
- Editor navigation and basic GameObject creation
- Simple conditionals and loops (if previously learned in CS course)
- Material color changes and basic lighting adjustments
- Using shortcuts (W, E, R keys) once familiar

## Suggested Checkpoints

### Formative checks during the unit
- "Show me your scene in both Scene and Game view" — reveals understanding of viewport
- "Run your script and show me the Console output" — confirms code is running
- "Explain what this variable/component does" — checks conceptual understanding
- "Make this change without looking at notes" — tests true mastery
- "Create a small example using today's concept" — formative mini-assessment

### Before moving to next section
- Can students apply the core concept in a new context?
- Can they debug a simple broken example?
- Can they explain the concept in their own words?

---

**Note for all instructors:**
- VGD uses Unity 6+ and C# 10+. Check that student Unity versions match.
- The STEM version is semester-length with tighter timelines — cut complexity and optional features before cutting core topics.
- For the year-long version, use the extra time to let students explore, build more ambitious projects, and deeply understand "why" before "how."
- Regularly checkpoint student understanding; adjust pacing in subsequent units based on prior unit performance.
