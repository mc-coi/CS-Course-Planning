# Video Game Design — Full Semester Course Materials

A complete, standards-aligned game development course for high school students. Built for an 18-week semester meeting 2 days per week (36 total class sessions), covering game design fundamentals and C# scripting in Unity from first game to polished capstone project.

## Unit Descriptions

### Unit 1 — Unity Interface & Game Design Thinking (Days 1–4)
Students are introduced to the Unity editor, game object hierarchy, scenes, and fundamental design concepts. Topics: navigating the editor, scene hierarchy, prefabs, game design documents, player experience, and the iterative design process. Ends with a design concept project.

### Unit 2 — C# Scripting Fundamentals (Days 5–8)
Deep dive into C# syntax and game scripting. Topics: variables, data types, loops, conditional logic, game loop mechanics, creating simple movement scripts, and understanding component-based architecture. Ends with a functional movement controller.

### Unit 3 — Physics & Collision Systems (Days 9–12)
Exploring Unity's physics engine and collision detection. Topics: rigidbodies, colliders, gravity, jumping mechanics, collision detection and response, physics materials, and event handling. Ends with a platformer mechanics project.

### Unit 4 — Player Controls & Input (Days 13–16)
Implementing responsive player input. Topics: input mapping, keyboard and mouse input handling, gamepad support, camera control, smooth movement, and game feel. Ends with a polished player controller project.

### Unit 5 — Game Design Principles & UI (Days 17–20)
Bridging design theory with implementation. Topics: challenge curves, feedback loops, progression systems, player motivation, UI design, menus, HUDs, and feedback systems. Ends with a complete UI system project.

### Unit 6 — 2D Game Development (Days 21–24)
Developing 2D games with sprites and animations. Topics: sprite management, tile maps, animation state machines, parallax scrolling, layer ordering, and 2D-specific physics. Ends with a small 2D platformer or puzzle game.

### Unit 7 — Game Systems & Enemy AI (Days 25–28)
Designing complex game systems and AI. Topics: scoring systems, health and damage mechanics, enemy behavior, state machines, AI pathfinding, difficulty scaling, and emergent gameplay. Ends with an intelligent enemy system project.

### Unit 8 — Capstone Project & Game Industry (Days 29–36)
Capstone unit applying all skills. Topics: project planning, complete game development, multiple levels, polished UI, sound integration, playtesting and feedback, game industry practices, professional development, and portfolio building. Ends with student-designed polished games and course reflection.

## File Types Explained

### Daily Lesson Plans (Word & PDF)
One file per class session. Each lesson plan follows a consistent format:
- **Learning Objectives** — what students will be able to do by the end
- **Warm-Up Activity** — brief review or hook activity
- **Direct Instruction** — new concept with code examples
- **Guided Practice** — teacher-led coding activity
- **Independent Practice** — student coding exercises
- **Exit Ticket** — reflection or formative assessment


### Unit Assessments (Word and PDF)
Each unit includes a comprehensive assessment in three formats:
- **.docx / .pdf** — printable version for traditional testing

Assessment question types include multiple choice, true/false, code analysis, write-the-code, identify-the-error, and short answer.

### Markdown Study Guides (.md)
One comprehensive study guide per day per unit. Each guide includes:
- Unit overview and learning targets
- Day-by-day key concepts with annotated code examples
- Practice problems at three tiers (Beginner / Intermediate / Challenge)
- Hint and answer collapsibles (`<details>` blocks)
- 15-problem practice bank
- Common mistakes table
- 10 assessment prep Q&A pairs
- Vocabulary and quick reference section

These are designed for student self-study, review, and reference. They render well in GitHub, VS Code, and most Markdown viewers.

## Standards Alignment

All materials are aligned to the **Tennessee Video Game Design** standards. The full mapping is in `TN_Video_Game_Design_Full_Semester_Standards_Alignment.pdf` at the root of this folder. Key standard areas covered:

- Game design thinking and iterative development
- C# programming fundamentals
- Game engine architecture and workflow
- Physics and collision systems
- Player input and game feel
- UI design and user experience
- 2D and 3D game development
- Game systems design and AI behavior
- Professional game development practices

## How to Use These Materials

**For daily instruction:** Open the `Word/Day N.docx` (or the matching PDF) for the current class session. Each lesson is self-contained and designed for a 90-minute period.

**For assessments:** Print the `.pdf` version from the `PDF/` folder.

**For student reference:** Share the unit's `.md` study guide — it's the go-to resource for review, homework help, and test prep.

**For sub plans:** The PDF lesson plans are fully self-contained. A substitute can run any lesson directly from the PDF with no additional context.
